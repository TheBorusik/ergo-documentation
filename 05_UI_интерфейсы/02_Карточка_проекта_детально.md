Примечание про раздел Закупки:
```

## ④ Снабжение — ЛИНИЯ ПРОЦЕССА (степпер), не аккордеон
```
Статус задачи меняется по шагам. Показываем как ЛИНИЮ:

┌─ Воздуховоды П1 → Поставщик А ──────────────────────┐
│ ①Запрос ─ ②Цены ─ ③Согл. ─ ④Оплата ─ ⑤Сроки ─ ⑥Доставка│
│    ✓        ✓       ●         ○         ○        ○    │
├──────────────────────────────────────────────────────┤
│ Текущий шаг: СОГЛАСОВАНИЕ                             │
│   цены получены, ждём решения менеджера               │
│   [Отправить на согласование менеджеру]              │
├──────────────────────────────────────────────────────┤
│ Позиции: Воздуховод ×14, Отвод ×8       [+ выбрать]   │
└──────────────────────────────────────────────────────┘

Линия статусов сверху (пройдено ✓ / текущий ● / впереди ○).
Клик на шаг → детали шага (что сделано: какой запрос, какие цены).
Для ТЕКУЩЕГО шага — кнопка действия.
Действия по шагам:
  Запрос → [Отправить] (модалка: email, шаблон)
  Цены → [Загрузить] (модалка: файлы, разметка, маппинг)
  Согласование → [На согласование] → менеджер [Согласовать]
  Оплата → [В оплату] → бухгалтер [Оплачено]
  Сроки → [Указать дату готовности]
  Доставка → [Оформить] (склад) → [Доставлено]

Меняется статус задачи → линия продвигается.
Наглядно ГДЕ в процессе + что делать сейчас.
```

Вертикальная линия событий с датами.

ИСТОРИЯ ГРУППЫ (агрегат — по помещению/задаче):
  "Созданы 10 позиций → переданы в снабжение →
   8 закуплено, 2 ждут → 5 доставлено..."
  линия процесса для пачки позиций
  (прогресс группы во времени)

ГДЕ показывать:
  - клик на позицию → её timeline (модалка/панель)
  - отдельный раздел/вкладка "История" проекта (все события)
  - лента событий в Обзоре (последние)

Данные: position_history (event, date, from→to, кто)
  → строится timeline
```

## Обновление модели — история
```
position_history (лог событий позиции)
  id, project_position_id,
  event_type (created/ready/purchased/delivered/installed/...),
  from_status, to_status,
  subtask_id (в какой задаче произошло),
  detail (поставщик, склад, цена...),
  changed_by, changed_at

→ timeline позиции = записи по ней
→ timeline группы = агрегат по нескольким позициям
→ лента проекта = все записи проекта
```

## ③ Доска — канбан задач по этапам
```
Раздел "Доска" (сверху в этапах):
  колонки = этапы, карточки = задачи
  на карточке — наглядный текущий статус задачи

┌─Проектир.─┬─Снабжение──┬─Работы──┬─Документы─┐
│┌─────────┐│┌──────────┐│         │           │
││Спец.пом1││││Воздух.П1 ││        │           │
││✓ готова ││││🛒 закупка││        │           │
│└─────────┘│└──────────┘│         │           │
│┌─────────┐│┌──────────┐│         │           │
││Спец.пом2││││Клапаны   ││        │           │
││⏳ в раб  ││││⏳ согл.  ││        │           │
│└─────────┘│└──────────┘│         │           │
└───────────┴────────────┴─────────┴───────────┘

Карточка задачи: название + текущий статус (цветной бейдж).
Клик → деталь задачи.
Задачи НЕ перетаскиваются между колонками (они в своих этапах).
Это ОБЗОРНЫЙ вид всех задач проекта разом.
```

## ④ Timeline агрегата (группы позиций) — оставляем
```
Для группы позиций — линия процесса (полезно):
  "10 позиций созданы → 8 закуплено → 5 доставлено → 2 установлено"

Показывает прогресс ПАЧКИ позиций во времени:
  по помещению, по спецификации, по задаче

Где: в Обзоре (агрегат проекта) или срез по помещению.
Визуал: линия с точками-этапами + счётчики на каждом.

┌─ Помещение 1 — процесс ────────────────────────┐
│ Создано ─── Закуплено ─── Доставлено ─── Монтаж  │
│   10          8             5            2       │
│   ●━━━━━━━━━━━●━━━━━━━━━━━━━●━━━━━━━━━━━━●        │
└─────────────────────────────────────────────────┘

Наглядно где пачка позиций в процессе.
```

═══════════════════════════════════════════════════
ИТОГ ПО РАЗДЕЛАМ КАРТОЧКИ (финал)
═══════════════════════════════════════════════════
```
РАЗДЕЛЫ:
  Обзор        — дашборд (метрики + агрегат-timeline по группам)
  ─ этапы ─
  Доска        — канбан задач (обзорный, все задачи)
  Проектирование — список задач + деталь
  Снабжение    — список задач + деталь (гориз. степпер)
  Работы       — список задач + деталь
  Документы    — список задач + деталь
  ─ данные ─
  Позиции      — все позиции (клик → история timeline)
  КП           — структура КП
  Файлы        — вложения
  История      — все события проекта (опц., для наглядности)

Шапка: название, заказчик, договор, срок, прогресс, [⋯]
  (без этапа проекта)
```
