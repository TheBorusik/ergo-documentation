# Раздел Поставщики (05/18) — история закупок

Решение 2026-07-22. Раздел строится из СНИМКА закупок: при оплате
закупки строки позиций фиксируются неизменяемо («как печать»),
раздел читает только их.

═══════════════════════════════════════════════════
СОКРАЩЕНИЕ РАСТУЩЕГО СПИСКА (2026-07-23, запрос пользователя)
═══════════════════════════════════════════════════
```
ВАЖНО (уточнение пользователя): фильтр периода и «неактуальные» НЕ в
агрегированном срезе «По позициям» (там бессмысленны — это сводка), а
при ОТКРЫТИИ номенклатуры (история цен по поставщикам). Срез «По
позициям» — чистый список (поиск + фасеты + пагинация).

В ИСТОРИИ ЦЕН НОМЕНКЛАТУРЫ (открытая позиция) — два механизма:

1. «НЕАКТУАЛЬНА» У КОНКРЕТНОГО ПОСТАВЩИКА — кнопка в строке (строка =
   поставщик+закупка). Помечает позицию устаревшей У ЭТОГО ПОСТАВЩИКА
   (напр. он больше её не возит / цена не актуальна). Строки этого
   поставщика по данной номенклатуре СКРЫВАЮТСЯ по умолчанию; тумблер
   «Показать неактуальные» возвращает их ПРИГЛУШЁННЫМИ («· неактуальна»)
   с кнопкой «Вернуть». «Мин»-цена считается только среди АКТУАЛЬНЫХ.
   Модель: purchase_archive (nomenclature_id, supplier_id, archived_at);
   метод SupplierNom.SetArchived {NomenclatureId, SupplierId, Archived}.
   SupplierPurchases.Query: флаг IsArchived per-строка; при NomenclatureId
   и без IncludeArchived архивные пары скрываются (в карточке поставщика
   его историю НЕ режем).

2. ФИЛЬТР ПО ПЕРИОДУ — DateFrom/DateTo (по purchased_at). В истории цен
   номенклатуры и в карточке поставщика (обе истории растут). UI — два
   поля-даты + сброс (компонент PeriodFilter).
```

═══════════════════════════════════════════════════
★ order_items — НЕИЗМЕНЯЕМЫЙ СНИМОК строк закупки
═══════════════════════════════════════════════════
```
Проблема: позиции проекта (project_positions) живут дальше по
конвейеру и меняются (статус, дробление, удаление). Закупка должна
зафиксировать «как было на момент оплаты» — документ.

order_items (пишется ТОЛЬКО в Orders.Pay, не меняется никогда):
  id
  order_id            FK→orders (к какой закупке)
  supplier_id         снимок поставщика (фильтр без join)
  nomenclature_id     FK→element_positions (ключ истории/подбора)
  project_position_id  происхождение (ссылка, НЕ источник данных)
  name, article       снимок наименования/артикула
  quantity, unit      снимок кол-ва и единицы
  price               цена ЗА ЕДИНИЦУ (снимок)
  currency            валюта (заложена сразу; пока RUB)
  line_total          сумма строки (снимок, не вычисляем)
  purchased_at        дата закупки (снимок из order.paid_at)

Денормализация (supplier_id, purchased_at, name…) намеренна —
история read-heavy, запросы без join. Immutable: нет update/delete.
orders дополнен полем currency.

supplier_prices (цены из прайса в задаче снабжения) остаются —
это ПРЕДЛОЖЕНИЯ; order_items — ФАКТ закупки.
```

═══════════════════════════════════════════════════
API (все с пагинацией {PageSize, Offset} + Total, ✓ реализованы)
═══════════════════════════════════════════════════
```
ErgoArea.Suppliers.Query { Search?, PageSize, Offset }
  → поставщики, У КОТОРЫХ БЫЛИ ЗАКУПКИ (distinct supplier_id из
    order_items) + агрегаты: OrdersCount, ItemsCount, TotalAmount,
    LastPurchaseAt. + Total. Сортировка по последней закупке.

ErgoArea.SupplierPurchases.Query
  { SupplierId?, NomenclatureId?, Search?, PageSize, Offset }
  → строки order_items (снимок) с фильтрами, сортировка по дате desc.
    Универсальный: вкладка поставщика (SupplierId) / история
    номенклатуры (NomenclatureId). + Total.

ErgoArea.SupplierPrices.Latest { Search?, PageSize, Offset }
  → срез «по позициям»: номенклатура → последняя цена (макс. дата) +
    поставщик + дата + PurchasesCount + SuppliersCount. + Total.

Все фильтры опциональны (как Subtasks.Query). Пагинация —
PageSize/Offset/Total (как каталог номенклатуры).
```

═══════════════════════════════════════════════════
UI — контейнер с двумя срезами
═══════════════════════════════════════════════════
```
Раздел = КОНТЕЙНЕР вкладок (05/11 п.11): [Список·pinned] + вкладки.
kinds: supplierList / supplier / supplierNom.
URL: /suppliers | /suppliers/supplier/{id} | /suppliers/nom/{id}

Список — переключатель среза (как «Списком/По КП» в Позициях):
  По поставщикам → Suppliers.Query: Поставщик | ИНН | Закупок |
    Позиций | Сумма | Последняя. Клик → ВКЛАДКА поставщика с его
    закупками (SupplierPurchases.Query SupplierId): Номенклатура
    (ссылка на историю) | Артикул | Кол-во | Цена/ед | Сумма |
    Дата | Заказ. Поиск + пагинация.
  По позициям → SupplierPrices.Latest: Номенклатура | Артикул |
    Последняя цена | Поставщик | Дата | История (N зак./M пост.).
    Клик → ВКЛАДКА истории номенклатуры (SupplierPurchases.Query
    NomenclatureId): все закупки по всем поставщикам, по дате desc,
    МИНИМАЛЬНАЯ цена подсвечена бейджем «мин».
```
