# Полное движение проекта — сценарий от начала до конца

Прослеживаем проект "Завод сыра" через все экраны и действия.
Кто, где, что нажимает.

═══════════════════════════════════════════════════
СТРУКТУРА НАВИГАЦИИ (исправлено)
═══════════════════════════════════════════════════
```
Левое меню:
  📊 Дашборд        ← ГЛАВНЫЙ экран (метрики, суммы, обзор)
  📁 Проеты
      ├─ Список проектов   ← основной вид (таблица)
      └─ Канбан            ← отдельный вид (доска по этапам)
  📦 Номенклатура
  🏭 Поставщики
  📋 Справочники
  ⚙️ Настройки
```

═══════════════════════════════════════════════════
ЭКРАН 0 — ДАШБОРД (главный, вход в систему)
═══════════════════════════════════════════════════
```
Открывается при входе. Обзор всего.

┌─────────────────────────────────────────────────┐
│ Дашборд                                          │
├─────────────────────────────────────────────────┤
│ [12 проектов] [3 в снабжении] [2 срочных]        │  ← карточки-метрики
│ [Сумма в работе: 45 млн] [Просрочено: 2]         │
├─────────────────────────────────────────────────┤
│ Проекты по этапам (мини-канбан или диаграмма):   │
│   Продажа 3 | Проектир. 2 | Снабжение 3 | ...     │
├─────────────────────────────────────────────────┤
│ Мои задачи        │ Последние события            │
│  - Согласовать... │  - Оплачен заказ П1          │
│  - Проверить...   │  - Доставлены воздуховоды     │
└─────────────────────────────────────────────────┘

Метрики: кол-во проектов, суммы (план/факт), просрочки,
распределение по этапам, мои задачи, лента событий.
```

═══════════════════════════════════════════════════
ЭТАП 1 — ПРОДАЖА (менеджер)
═══════════════════════════════════════════════════

## Шаг 1.1 — Создание проекта
```
ГДЕ: Проекты › Список проектов
ДЕЙСТВИЕ: кнопка [+ Новый проект]
→ Модалка создания:
   Название: "Завод сыра — вентиляция"
   Заказчик: [выбор из контрагентов ▾] или [+ новый]
   Менеджер: [я]
   [Создать]
→ проект создан, статус "Продажа", открывается карточка проекта
```

## Шаг 1.2 — Формирование КП
```
ГДЕ: Карточка проекта › вкладка "КП"
ДЕЙСТВИЕ: строим структуру КП
   [+ Группа] → "Оборудование"
   внутри [+ Запись] → "Приточная установка П1", кол-во 1, цена
   повторяем: группы (Холод, Материалы, Работы...) + записи
→ КП готово (верхнеуровневое)
Может: импорт КП из Excel (загрузка готового)
```

## Шаг 1.3 — Выигрыш / договор
```
ГДЕ: Карточка проекта (шапка)
ДЕЙСТВИЕ: кнопка [Выиграли] (или [Проиграли])
   → статус: won, вводим № договора, срок
   [Передать в проектирование] → проект переходит на этап Проектирование
Проигранный: [Проиграли] → статус lost, проект в архиве (но в системе)
```

═══════════════════════════════════════════════════
ЭТАП 2 — ПРОЕКТИРОВАНИЕ (инженер)
═══════════════════════════════════════════════════

## Шаг 2.1 — Инженер видит проект
```
ГДЕ: Дашборд / Список проектов (фильтр "Проектирование")
ДЕЙСТВИЕ: открывает карточку проекта "Завод сыра"
Видит: КП (что продано), нужно наполнить позициями
```

## Шаг 2.2 — Создание спецификации
```
ГДЕ: Карточка проекта › вкладка "Спецификации"
ДЕЙСТВИЕ: [+ Спецификация]
   Название: "Спецификация помещение 1"
   → создалась подзадача-спецификация (stage=Проектирование)
```

## Шаг 2.3 — Наполнение позициями
```
ГДЕ: внутри спецификации
ДЕЙСТВИЕ: [+ Позиция]
   → поиск/выбор номенклатуры (Воздуховод 500х300...)
   → если нет — [Создать позицию] (модалка конструктора)
   → указываем количество: ×14
   → ВЫБИРАЕМ к какому КП относится: [Приточная П1 ▾]
   повторяем для всех позиций помещения
→ позиции созданы (status=spec_ready), привязаны к КП
```

## Шаг 2.4 — Завершение спецификации → в снабжение
```
ГДЕ: спецификация
ДЕЙСТВИЕ: [Завершить спецификацию]
   → позиции готовы к снабжению
   → [Передать в снабжение] (можно частями — каждую спецификацию)
Инженер может создавать спецификации параллельно.
Когда весь проект готов: [Готов к снабжению] в шапке
   → проект переходит на этап Снабжение
```

═══════════════════════════════════════════════════
ЭТАП 3 — СНАБЖЕНИЕ (снабженец)
═══════════════════════════════════════════════════

## Шаг 3.1 — Снабженец видит позиции
```
ГДЕ: Карточка проекта › вкладка "Позиции" (или "Задачи")
Видит: позиции со статусом "готовы к закупке" (из спецификаций)
```

## Шаг 3.2 — Создание задачи закупки (дробление)
```
ГДЕ: вкладка "Позиции"
ДЕЙСТВИЕ: выбирает позиции галочками (Воздуховод, Отвод...)
   → [Создать задачу снабжения]
   → Название: "Воздуховоды П1", выбрать поставщика (опц.)
   → задача создана (позиции ЗАНЯТЫ, эксклюзивность)
Дробит на несколько: "Воздуховоды", "Клапаны", "Крепёж"...
```

## Шаг 3.3 — Запрос цен поставщику
```
ГДЕ: Экран задачи снабжения
ДЕЙСТВИЕ: [Отправить запрос]
   → модалка: поставщик, email, шаблон, превью позиций
   → [Отправить] → email ушёл, статус "отправлен"
Можно отправить другому поставщику тоже (сравнить).
```

## Шаг 3.4 — Загрузка цен + маппинг
```
ГДЕ: Экран задачи снабжения
ДЕЙСТВИЕ: [Загрузить цены]
   → модалка: [+ Добавить файл] (можно несколько)
   → файлы парсятся (C#), показывается таблица
   → если формат новый: разметка колонок (клик)
   → МАППИНГ: наши ↔ поставщика (авто + подтверждение fuzzy)
   → [Сохранить цены]
→ цены сохранены, статус "цены получены"
```

## Шаг 3.5 — Согласование (менеджер)
```
ГДЕ: снабженец [Отправить на согласование]
   → у менеджера в "Мои задачи" появляется
МЕНЕДЖЕР: открывает, видит цены, [Согласовать] / [Отклонить]
   → статус "согласована"
```

## Шаг 3.6 — Оплата (бухгалтерия)
```
ГДЕ: снабженец [Передать в оплату]
   → у бухгалтера в задачах
БУХГАЛТЕР: [Оплачено] → статус "оплачена"
```

## Шаг 3.7 — Сроки и доставка
```
ГДЕ: Экран задачи снабжения
ДЕЙСТВИЕ: снабженец проставляет [Сроки готовности]
   → ждём → [Готово у поставщика]
   → [Оформить доставку]: выбор склада (проекта/общий)
   → [Доставлено] → позиции status=delivered
   → [Закрыть задачу] → status=done
→ позиции освобождаются, готовы к монтажу
→ в проекте видно: эти позиции "доставлены"
```

═══════════════════════════════════════════════════
ЭТАП 4 — МОНТАЖ И ПНР (прораб)
═══════════════════════════════════════════════════

## Шаг 4.1 — Создание задачи на работы
```
ГДЕ: Карточка проекта › вкладка "Позиции" (доставленные)
ДЕЙСТВИЕ: [Создать задачу работ]
   → Вид работ: [Монтаж воздуховодов ▾] (из справочника)
   → ВЫБОР позиций для работы (воздуховоды + крепёж + болты)
   → задача создана (позиции заняты)
```

## Шаг 4.2 — Выполнение
```
ГДЕ: Экран задачи работ
ДЕЙСТВИЕ: [В работу] → выполняют → [Выполнено]
   → позиции status=installed
Прораб может дробить работы (как снабженец).
```

## Шаг 4.3 — Акт (опционально)
```
ГДЕ: вкладка "Задачи" или отдельно "Акты"
ДЕЙСТВИЕ: [Создать акт] → выбор выполненных работ (одну/несколько)
   → акт оформлен (отдельно от статусов позиций)
```

═══════════════════════════════════════════════════
ЭТАП 5 — ДОКУМЕНТАЦИЯ (сотрудник)
═══════════════════════════════════════════════════

## Шаг 5.1 — Навешивание документов
```
ГДЕ: Карточка проекта › вкладка "Документы"
Видит: позиции (установленные), у каждой нужен документ
ДЕЙСТВИЕ: у позиции [Выбрать документ ▾]
   → выпадающий список действующих сертификатов/паспортов
   → выбирает (один документ можно на несколько позиций)
   → если нет: [+ Добавить документ] (модалка справочника)
→ все позиции получают документ
```

## Шаг 5.2 — Финальный документ
```
ГДЕ: вкладка "Документы"
ДЕЙСТВИЕ: [Сформировать сводный документ]
   → группировка по сертификату/паспорту → позиции
   → PDF-пакет для заказчика
→ [Готово к сдаче]
```

═══════════════════════════════════════════════════
ЭТАП 6 — СДАЧА (менеджер)
═══════════════════════════════════════════════════
```
ГДЕ: Карточка проекта (шапка)
ДЕЙСТВИЕ: [Отправить заказчику] → финальный документ
   → приёмка → [Проект принят] → финальная оплата
   → [Завершить проект] → status=completed
→ проект в архиве завершённых
```

═══════════════════════════════════════════════════
СВОДКА — КТО ГДЕ ЧТО НАЖИМАЕТ
═══════════════════════════════════════════════════
```
Менеджер:  создаёт проект, КП, выигрыш, сдача
Инженер:   спецификации, позиции, передача в снабжение
Снабженец: задачи закупки, запросы, цены/маппинг, доставка
Менеджер:  согласование цен
Бухгалтер: оплата
Прораб:    задачи работ, выполнение, акты
Сотрудник: документы на позиции, сводный документ

Проект двигается по этапам кнопками "Готов к следующему".
Позиции текут по конвейеру, меняя статус от задач.
Задачи создаются на каждом этапе (выбор позиций).
```

═══════════════════════════════════════════════════
КЛЮЧЕВЫЕ ДЕЙСТВИЯ (паттерн везде одинаковый)
═══════════════════════════════════════════════════
```
1. Выбрать позиции (галочками)
2. Создать задачу этапа (дробление)
3. Работать с задачей (свои действия этапа)
4. Завершить задачу → позиции меняют статус
5. Позиции доступны следующему этапу

Этот паттерн ОДИНАКОВ на снабжении, работах, документации.
Разное только "работа с задачей" (шаги процесса).
```
