# Раздел Контрагенты (Companies) — справочник организаций

Перенесено из первой реализации Ergo (ergo-old/companies) в формат наших доков.
Единый справочник организаций: заказчики, поставщики, подрядчики.
Пункт меню «Поставщики» из 05/00 покрывается этим разделом (шире: все контрагенты).

═══════════════════════════════════════════════════
НАЗНАЧЕНИЕ И КЛЮЧЕВОЕ
═══════════════════════════════════════════════════
```
ОДНА сущность на все организации (Contractor/Company):
  роль различает: Заказчик (Client) | Поставщик (Supplier) | Подрядчик (Contractor)

★ АВТОЗАПОЛНЕНИЕ РЕКВИЗИТОВ — главная фишка раздела:
  поиск организации в ФНС или Checko по ИНН/ОГРН/названию
  → выбрал из результатов → форма заполнилась реквизитами
  → руками вводим только контакты и роль.

Используется всей системой:
  Проект.customer_id      → контрагент-заказчик
  Позиция.supplier_id     → контрагент-поставщик
  Order.supplier_id       → контрагент-поставщик
  SupplierRequest.supplier_id → запросы цен
```

## Действия
```
[+ Добавить компанию] → модалка добавления (ниже)
✎ редактировать      → форма редактирования (та же модалка с данными)
🗑 удалить            → диалог подтверждения:
   "Удалить компанию? Это действие нельзя отменить" +
   карточка (название, ИНН) + "Будут удалены все связанные данные"
   ВОПРОС: запретить удаление, если контрагент используется
   (проекты/позиции/order) — мягкое is_active=false вместо удаления?
```

```
Тип организации (ЮЛ | ИП | НР) — readonly, приходит из ФНС/Checko.
Роль выбирается вручную (Заказчик | Поставщик | Подрядчик).
После выбора в поиске ФНС/Checko автозаполняются:
  Name, FullName, INN, OGRN, Type, RegisterDate, Address,
  Status, MainActivity. Контакты — вручную.
```

═══════════════════════════════════════════════════
МОДЕЛЬ (объединение Contractor + Company)
═══════════════════════════════════════════════════
```
Contractor (= Company из ergo-old, база — она богаче):
  id: int
  role: enum (customer | supplier | contractor)   ← роль в системе
  name: string             (краткое, "ООО Вент")
  full_name: string        (полное юридическое)
  inn: string
  kpp: string?
  ogrn: string
  org_type: enum (ЮЛ | ИП | НР)?     (из ФНС, readonly)
  register_date: date?               (дата регистрации)
  fns_status: string?                (статус из ФНС: "Действующее"...)
  address: string
  main_activity: string?             (основная деятельность)
  contact_person: string?
  phone: string?
  email: string?
  website: string?
  note: text?
  is_active: bool                    (архивные скрывать)
  created_at, updated_at

Отличия от старых моделей:
  - ergo-docs Contractor.type (customer|supplier|both) → role
    (подрядчик добавился; "both" — открытый вопрос ниже)
  - ergo-old Company: +Role=Contractor, +ОГРН/тип/статус/дата из ФНС
  - сервер ergo-old возвращал "Adress" (опечатка) — клиент нормализовал
    в Address. В новом бэке — Address сразу.

CompanySearchResult (результат поиска ФНС/Checko — не хранится):
  type, inn, ogrn, kpp?, name, full_name, status, address,
  main_activity, registration_date, okved?, region_code?,
  liquidation_date?, source?

CheckoMeta { balance: int, today_requests: int }
```
