# Раздел Продажи (ProjectSales) — воронка сделок (пресейл)

Перенесено из первой реализации Ergo (ergo-old/projectsales) в формат наших доков.
Закрывает отложенную тему B.1 из 01/07 («Продажи для руководителя»).

═══════════════════════════════════════════════════
НАЗНАЧЕНИЕ И МЕСТО В СИСТЕМЕ
═══════════════════════════════════════════════════
```
СДЕЛКА (ProjectSale) — потенциальный проект ДО выигрыша.
Это CRM-воронка: прогнозирование, взвешенные суммы, фазы переговоров.

НЕ путать с этапом «Продажа/КП» (04/00):
  Сделка (этот раздел)  — ДО проекта: переговоры, вероятности, прогноз
  Этап Продажа (КП)     — содержимое проекта: блоки, записи, позиции

СВЯЗЬ С КОНВЕЙЕРОМ (предложение, вопрос ниже):
  Сделка Won → создаётся Project (Сделка.project_id заполняется)
  Сделка Lost → остаётся в системе (статистика проигранных — 01/00
    "Проигранные тоже в системе (lost)")
  → сделка = преджизнь проекта, конвейер начинается после Won.

ПРОГНОЗ (ключевая механика):
  вероятность запуска (0/30/60/90/100%) ×
  вероятность подписания (0/30/60/90/100%) = итоговый %
  Вес = Стоимость × итоговый %  → взвешенная сумма воронки
  Σ по воронке = прогноз выручки для руководителя.
```

СПРАВОЧНЫЕ ЗНАЧЕНИЯ (enum + метки RU)
═══════════════════════════════════════════════════
```
СТАТУС СДЕЛКИ (ProjectSaleStatus):
  InProgress — В работе    (цвет brand-primary)
  Stopped    — Остановлен  (warning)
  Won        — Победа      (success)
  Lost       — Поражение   (error)

ФАЗА ПРОДАЖ (SalesPhase) — воронка:
  RequestAnalysis — Анализ запроса
  BudgetOffer     — Бюджетное КП
  FirmOffer       — Твёрдое КП
  Negotiations    — Переговоры
  Decision        — Решение
  Contract        — Договор

ВИД УСЛУГИ (ServiceType) — МНОЖЕСТВЕННЫЙ выбор (чипы):
  Supply      — Поставка       (тёмно-синий #1a365d)
  Engineering — Проектирование (тёмно-фиолетовый #44337a)
  Jobs        — Работы         (терракотовый #9a3412)
  Service     — Сервис         (тёмно-зелёный #065f46)

ВЕРОЯТНОСТИ (% запуска, % подписания) — дискретные:
  0% (error) | 30% (warning) | 60% (#eab308) | 90% (#84cc16) |
  100% (success)
```

ЭКРАН СПИСКА
═══════════════════════════════════════════════════
```
┌─ Проектные продажи                       [+ Создать проект]────┐
│ Создание и прогнозирование проектами                           │
│                                                                │
│ ┌─────────────┬───────────┬────────────────┬────────────────┐  │
│ │📁 Всего     │📈 В работе│₽ Общая         │⚖ Общий вес     │  │ ← summary
│ │  проектов N │     N     │  стоимость Σ   │  (взвешенный Σ)│  │
│ └─────────────┴───────────┴────────────────┴────────────────┘  │
│                                                                │
│ [🔍 Поиск: название, организация, место…] [Все статусы▾][Поиск]│
├────────────────────────────────────────────────────────────────┤
│ИНН│Организация│Наименование│Место│Услуги│Дата подп.│Статус│Фаза│
│Продавец│%зап.│%подп.│Итого%│Стоимость│Вес│⋯                    │
│  (строка кликабельна → карточка сделки /projectsales/{id})     │
├────────────────────────────────────────────────────────────────┤
│                      [Загрузить ещё]                           │ ← курсорная
└────────────────────────────────────────────────────────────────┘   пагинация

Услуги — цветные чипы (несколько на сделку).
Статус и фаза — цветные бейджи. Проценты подкрашены по шкале.
Вес = Cost × TotalPercentage (считает сервер, приходит в модели).
Пагинация: PageSize + LastId/LastSortValue (курсор), кнопка
  "Загрузить ещё" пока NextPage=true.
Summary приходит С СЕРВЕРА в ответе Query (по всей выборке,
  не по загруженной странице).
```

## Действия
```
[+ Создать проект] → модалка создания (ниже)
клик по строке     → карточка сделки (редактирование)
🗑 в строке         → диалог подтверждения удаления
Поиск: Enter или кнопка → серверный поиск (Filter.Search)
Фильтр статуса → перезапрос с Filter.Status (сброс курсора)
```

МОДЕЛЬ
═══════════════════════════════════════════════════
```
ProjectSale (сделка):
  id: int
  project_id: FK→Project?     (заполняется при Won — вопрос ниже)
  company_id: FK→Contractor   (организация-заказчик)
  name: string                (наименование)
  location: string            (местоположение объекта)
  service_types: ServiceType[] (мультивыбор)
  expected_signing_date: date
  status: enum (in_progress | stopped | won | lost)
  sales_phase: enum (request_analysis | budget_offer | firm_offer |
                     negotiations | decision | contract)
  user_id: FK→User            (продавец/ответственный)
  launch_percentage: int      (0|30|60|90|100)
  contract_signing_percentage: int (0|30|60|90|100)
  total_percentage: int       (вычисляемое: launch×signing/100)
  cost: decimal               (стоимость)
  weight: decimal             (вычисляемое: cost × total%)
  comment: text?
  created_at, updated_at

В ответах сервер разворачивает связи:
  Company { Id, INN, Name }   (краткая карточка контрагента)
  User { UserId, FirstName, LastName, ... }  (продавец)
  ⚠ в ergo-old в User сделки приезжали поля волейбольного профиля
    (Height, Position…) — наследие VolleyDzen, в новую модель НЕ берём.

ProjectSalesSummary (агрегат для карточек):
  total_projects, projects_in_progress, total_cost, total_weight
```
