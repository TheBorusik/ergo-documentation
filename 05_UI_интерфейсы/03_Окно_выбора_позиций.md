# Окно выбора позиций (детально) — дерево + корзина

Единая модалка для всех этапов (снабжение, работы, документация).
Решает проблему "размазанного выбора" — корзина справа собирает всё.

МАКЕТ
═══════════════════════════════════════════════════
```
┌─ Выбор позиций для задачи снабжения ──────────────────────────┐
│ Вид: [По спецификации▾] [По КП]      Поиск: [______] 🔍       │
├────────────────────────────────┬──────────────────────────────┤
│ ДОСТУПНЫЕ ПОЗИЦИИ               │ ВЫБРАНО (5)                   │
│                                 │                               │
│ ▾ Помещение 1                   │ Воздуховод 500х300 ×14   [✕] │
│    Воздуховод 500х300 ×14  [→]  │ Отвод 500х300 ×8         [✕] │
│    Отвод 500х300 ×8        [→]  │ Воздуховод 400х200 ×20   [✕] │
│    Клапан 400х400 ×2       [→]  │ Клапан 400х400 ×2        [✕] │
│    ⊘ Вентилятор (проектир.)     │ Отвод 400х200 ×4         [✕] │
│ ▾ Помещение 2                   │                               │
│    Воздуховод 400х200 ×20  [→]  │                               │
│    Отвод 400х200 ×4        [→]  │                               │
│ ▾ Материалы                     │                               │
│    Воздуховоды оц ×720     [→]  │                               │
├────────────────────────────────┴──────────────────────────────┤
│ Выбрано 5 позиций                    [Отмена] [В задачу]       │
└─────────────────────────────────────────────────────────────────┘
```

═══════════════════════════════════════════════════
СТРУКТУРА
═══════════════════════════════════════════════════
```
ДВЕ ПАНЕЛИ:

СЛЕВА — дерево ДОСТУПНЫХ позиций
  группировка (переключаемая): По спецификации / По КП
  [→] перекинуть позицию в корзину
  ⊘ недоступные — серые, видны, БЕЗ кнопки (видно почему)
  поиск сверху

СПРАВА — КОРЗИНА выбранного (плоский список)
  всё выбранное в одном месте (из разных групп)
  [✕] убрать обратно
  счётчик "Выбрано (N)"

→ решает "размазанный выбор": итог всегда виден справа
```

═══════════════════════════════════════════════════
ГРУППИРОВКА (переключатель вида)
═══════════════════════════════════════════════════
```
ПО СПЕЦИФИКАЦИИ (по умолчанию):
  ▾ Помещение 1 → позиции
  ▾ Помещение 2 → позиции
  ▾ Материалы → позиции
  (spec_subtask_id)

ПО КП:
  ▾ Приточная П1 → позиции
  ▾ Материалы → позиции
  (kp_item_id)
  → видно как соотносится с КП

Переключение не меняет выбор (корзина сохраняется).
```

═══════════════════════════════════════════════════
ДОСТУПНОСТЬ ПОЗИЦИЙ (по этапу)
═══════════════════════════════════════════════════
```
Позиция доступна (можно [→]) если:
  status = нужный для этапа
    снабжение → spec_ready
    работы → delivered
    документация → installed
  AND current_subtask_id = null (свободна)

Недоступные — серые, видны (человеческий фактор):
  причина подписана (проектируется / в закупке / занята)
```

═══════════════════════════════════════════════════
ДРОБЛЕНИЕ ПРИ ВЫБОРЕ (взять часть)
═══════════════════════════════════════════════════
```
При [→] можно взять ЧАСТЬ количества:

Воздуховод ×14 [→] →
  ┌─ Сколько взять? ──────┐
  │ Доступно: 14          │
  │ Взять: [10]           │
  │      [Отмена] [ОК]    │
  └───────────────────────┘
  → в корзину ×10
  → осталось ×4 доступно (слева)

Если взял часть → позиция дробится:
  создаётся Position ×10 (в задачу) + остаётся ×4
  (parent_position_id — связь, общий kp_item)

По умолчанию берётся ВСЁ количество (без спроса),
дробление — опция (клик на количество перед [→]).
```

═══════════════════════════════════════════════════
ПРИМЕНЕНИЕ НА ЭТАПАХ
═══════════════════════════════════════════════════
```
Снабжение: доступны spec_ready → в задачу закупки
Работы:    доступны delivered → в задачу работ
Документация: доступны installed → в задачу документов

Одна модалка, параметр — этап (определяет доступность).
Группировка и корзина одинаковы везде.
```

API
═══════════════════════════════════════════════════
```
GET /api/projects/{projectId}/positions/available
  Параметры:
    stage: enum (supply|works|docs)  (определяет доступность)
    group_by: enum (spec|kp)          (группировка)
  Ответ: 200
    {
      groups: [
        { key: int, name: string,     (спецификация или kp_item)
          positions: [ PositionDetail + {
            selectable: bool,
            reason: string?            (почему недоступна)
          } ] }
      ]
    }

POST /api/subtasks/{id}/positions
  (добавить выбранные — с опц. дроблением)
  Тело:
    { items: [ { position_id: int, quantity?: decimal } ] }
    (quantity меньше полного → дробление)
  Ответ: 200 { items: PositionDetail[] }
  Побочно:
    полное кол-во → position.current_subtask_id = задача
    частичное → split: новая Position(qty) + остаток
```

## Дробление — отдельная иконка + клонирование при подтверждении

```
Рядом со стрелкой [→] — иконка дробления [⑃]:
  Воздуховод ×14   🟢 готова   [⑃] [→]
  [→]  — всё количество в корзину
  [⑃]  — выбор части: "взять 10 из 14"

КОГДА КЛОНИРОВАТЬ (выбрано: Вариант А):
  дробление в окне — ВИРТУАЛЬНОЕ (черновик выбора)
  реальный split происходит при [В задачу] (подтверждение)

  Пока окно открыто — ничего в БД не создано:
    ✓ отмена бесплатна (закрыл окно → ничего не изменилось)
    ✓ можно передумать, вернуть, переразбить

  При [В задачу]:
    позиция ×14 → split: Position ×10 (в задачу) + Position ×4 (остаток)
    parent_position_id связывает, общий kp_item_id

Отмена дробления в окне:
  [✕] в корзине → часть возвращается, слева снова ×14
```

## Подсветка раздробленного
```
Раздробленная позиция помечена визуально:
  СЛЕВА:   "Воздуховод ×4 (было 14)"  significant значок дробления
  КОРЗИНА: "Воздуховод ×10 (часть)"   значок части

Видно что позиция разделена (не потерялась, не задвоилась).
```

## Статус позиций — показывать (важно)
```
Каждая позиция — статус (что с ней):

▾ Помещение 1
   Воздуховод ×14   🟢 готова        [⑃][→]
   Клапан ×2        🟢 готова        [⑃][→]
   Вентилятор ×1    ⚪ проектируется  (недоступна, серая)
▾ Помещение 2
   Отвод ×4         🟡 в закупке     (занята, недоступна)

Статус: цвет + подпись.
  🟢 готова (доступна) — есть [⑃][→]
  ⚪ проектируется (ещё не готова) — серая
  🟡 в закупке/занята — серая
  ✅ доставлена / 🔨 монтаж — для других этапов

Видно ПОЧЕМУ доступна/недоступна (человеческий фактор).
```

## Массовый выбор группы
```
Кнопка "вся группа" рядом с заголовком группы:
  ▾ Помещение 1                    [→ всё]
  → все доступные позиции группы в корзину разом
Отмена: [✕] на каждой в корзине (или вернуть группой).
```

## Обновление API (дробление при подтверждении)
```
POST /api/subtasks/{id}/positions
  Тело:
    { items: [ { position_id: int, quantity?: decimal } ] }
    quantity < полного → split при обработке
  Ответ: 200 { items: PositionDetail[], split: PositionDetail[] }
  Побочно:
    полное → current_subtask_id = задача
    частичное → split (клон qty в задачу + остаток),
                parent_position_id, position_history

  Дробление применяется ЗДЕСЬ (при подтверждении),
  не раньше. До этого — виртуальный выбор на фронте.
```

## Проблема повторного дробления
```
Раздробил ×14 → ×10 в задачу (БД: Position ×10 + Position ×4)
Потом снова открыл окно → добавил ×4
→ две записи "Воздуховод" (×10 + ×4) в одной задаче (дубли)

Некрасиво: по сути те же воздуховоды, разделены зря.
```

## Решение — объединение (merge)
```
ПОДСКАЗКА + КНОПКА (выбрано):
  система замечает в задаче части одной номенклатуры →
  подсвечивает "можно объединить" → [Объединить]

При [В задачу] после объединения:
  добавленную ×4 УДАЛИТЬ,
  у существующей ×10 → количество стало ×14
  (обратный split = merge)

Не автоматически (вдруг специально раздельно),
но подсказка чтобы не пропустить дубли.
```

## Отображение дробления — ФИНАЛ: формат ×N (общее) + цветовая связь
```
КОЛИЧЕСТВО — единый формат ×N (как везде в интерфейсе):
  обычная позиция:      Воздуховод ×14
  раздробленная часть:  Воздуховод ×4 (14)
    ×4 — эта часть, (14) — общее исходное количество

ЦВЕТОВАЯ СВЯЗЬ — пары одного дробления одним цветом:
  Воздуховод ×4 (14)  🔵
  Воздуховод ×10 (14) 🔵   ← та же синяя точка = одна пара

  Отвод ×5 (8)  🟢
  Отвод ×3 (8)  🟢         ← зелёная = другая пара

Цветная точка/метка связывает части.
Дроблений немного → цветов хватает (палитра ~8 цветов).

Отклонённые:
  дробь X/Y — не понравилось (не единообразно с ×N)
  половинки иконки (◧◨) — без чисел целого
  номер пары [дроб.N] — заменён цветом (нагляднее)
```

## Модель — merge
```
POST /api/subtasks/{id}/positions/merge
  (объединить части одной номенклатуры)
  Тело: { position_ids: int[] }  (части для слияния)
  Ответ: 200 PositionDetail  (объединённая)
  Побочно:
    оставить одну Position, сумма quantity,
    остальные удалить, parent_position_id очистить,
    position_history (merge)

Проверка дублей (для подсказки):
  GET /api/subtasks/{id}/positions
    → если несколько Position с одним position_id (номенклатура)
      → флаг can_merge в ответе
```

## Итоговое отображение в окне/задаче
```
Раздробленная позиция:
  дробь X/Y + [дроб.N] если несколько дроблений
  Воздуховод 10/14 [дроб.1]

Дубли в задаче:
  подсказка "можно объединить" + кнопка
  Воздуховод ×10  [Объединить → ×14]
  Воздуховод ×4
```
