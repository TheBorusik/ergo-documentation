# Раздел Справочники (системные)

Решение 2026-07-22, реализовано. Системные справочники живут в меню
СИСТЕМА → Справочники; номенклатурные — в своём разделе (05/16).

═══════════════════════════════════════════════════
НАЗНАЧЕНИЕ И УСТРОЙСТВО
═══════════════════════════════════════════════════
```
КОНТЕЙНЕР вкладок (единый паттерн 05/11 п.11):
  [Список · pinned] + вкладки открытых справочников
  URL: /dictionaries(/{dictKey})

Список — таблица справочников (название, описание), клик → вкладка.
Вкладка справочника — значения ЧЕРЕЗ ЧЕРНОВИК (единый механизм):
  правки/добавления/удаления копятся локально, изменённое
  подсвечивается (жёлтая рамка/полоска — lib/changed), dirty ● на
  вкладке, SaveGuard при закрытии, одна кнопка [Сохранить] шлёт
  ВЕСЬ черновик — diff считает СЕРВЕР (паттерн Kp.Save).

Строка значения: Название | Активно (чекбокс) | 🗑
Используемое значение удалить нельзя (сервер проверяет связи при
сохранении и возвращает ошибку) — деактивировать вместо удаления.
```

═══════════════════════════════════════════════════
СЕРВЕРНАЯ ЛОГИКА (образец — Works.Save, 5Б-шаги)
═══════════════════════════════════════════════════
```
ErgoArea.Works.Save { Items: [{ Id, Name, IsActive }] }
(Id < 0 — новые значения)

1. works          → SELECT все → существующие
2. Для отсутствующих в Items:
   subtask_works  → SELECT где work_id → ПРОВЕРКА использования
                    (используется → ошибка «деактивируйте»)
   works          → DELETE
3. Для каждой присланной:
   Id < 0         → works INSERT (name, is_active)
   иначе          → works UPDATE (name, is_active)
4. event_log      → INSERT 'dictionary_saved'

Возврат: Works.Query {All:true} (полный список — новый baseline)
```
