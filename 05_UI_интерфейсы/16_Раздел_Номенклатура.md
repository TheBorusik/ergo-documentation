(справочники, конструктор элементов с предпросмотром, каталог +

КЛЮЧЕВАЯ ИДЕЯ — три рабочих зоны в одной вкладке
═══════════════════════════════════════════════════
```
Процесс ядра (02/01 §3) — два этапа, три роли работы:
  1. СПРАВОЧНИКИ  (технолог): свойства, единицы, уточнения, категории
  2. ЭЛЕМЕНТЫ     (технолог): конструктор шаблонов (свойства + имя/артикул)
  3. КАТАЛОГ      (пользователь): создание позиций по шаблону + поиск

→ Глобальная вкладка «Номенклатура» с ВНУТРЕННИМИ вкладками
  (наша рекурсивная модель вкладок это уже умеет — как карточка проекта):

┌─ Номенклатура ─────────────────────────────────────────┐
│ [⌂ Каталог·pinned] [Элементы] [Справочники] [+позиция…]│ ← уровень 2
│ ┌─ внутри Справочников — свой уровень (рекурсия): ────┐ │
│ │ [Свойства·pinned][Уточнения][Единицы][Категории]    │ │ ← уровень 3
│ │ [Литералы]                                          │ │
│ └──────────────────────────────────────────────────────┘ │
└────────────────────────────────────────────────────────┘

Каталог закреплён (рабочая зона пользователя), Элементы и
Справочники — для технолога (под правами ERP.Nomenclature.Admin).
URL: /nomenclature/(catalog|elements|dicts/(properties|qualifiers|
units|categories|literals))
```

1. КАТАЛОГ (pinned) — позиции + фасетный поиск
═══════════════════════════════════════════════════
★ РЕАЛИЗОВАНО (2026-07-18) «как в маркетплейсах» (уточнение пользователя):
```
┌ КАТЕГОРИИ ┬ СВОЙСТВА (фасеты) ┬ РЕЗУЛЬТАТЫ ─────────────────┐
│ Все катег.│ Ширина  [число…]  │ [🔍 поиск]     [+ Позиция]  │
│ ▸Вентиляц.│ Высота  [число…]  │ Артикул│Название│Элемент    │
│  Крепёж   │ Металл:           │ …строки…                    │
│  …        │  ☑ оц.сталь    3  │                             │
│           │ Тип сечения:      │                             │
│           │  ☐ прямоуг.   11  │                             │
│           │  ☐ круглый     2  │ 1–20 из 154   ◀ стр.1 ▶     │
│           │ [Сбросить]        │                             │
└───────────┴───────────────────┴─────────────────────────────┘
Unit-свойство → ввод числа (точное совпадение);
Enum → чекбоксы значений со счётчиками позиций;
fixed-свойство элемента = значение всех его позиций (без двойного
счёта с явными значениями). Комбинация фильтров = AND (02/01 §5.3).
Пагинация ОБЯЗАТЕЛЬНА (позиций много): Limit/Offset + Total.
Компоненты переиспользуемые (FacetNav) — та же навигация в форме
добавления позиций в спецификацию (04/01 §2.3).
```
```
┌─────────────┬──────────────────────────────────────────┐
│ КАТЕГОРИИ    │ [🔍 Поиск] [Тип изделия▾] [+ Позиция]    │
│ Вентиляция   │ ФАСЕТЫ: [Металл: оц.сталь ✕][Ширина: 500]│
│ Вент.оборуд. │──────────────────────────────────────────│
│ Холод        │ Артикул         │ Название      │Элемент │

│ Монтажные    │ БЛ-М6-20-DIN…   │ Болт метрич…  │Болт     │
│ Электрика    │ …                                        │
│ Изоляция     │                                          │
│ Прочее       │                                          │
└─────────────┴──────────────────────────────────────────┘

Слева — плоское дерево категорий (VS Code sidebar-стиль, счётчики).
Справа — таблица позиций. ФАСЕТЫ строятся от выбранной категории:
  Тип изделия (фасет №1, reference 57) → фикс.свойства элемента →
  свойства позиций (Ширина, Металл…) — чипы над таблицей.
Клик по позиции → панель детали (свойства + где используется).
Этот же экран (без [+ Позиция]) потом встраивается в Проектирование
как модалка выбора номенклатуры (памятка 01/04 §10).
```

```
┌─ Новая позиция ────────────────────────────── ✕ ─┐
│ Элемент: [Воздуховод прямоугольный ▾ 🔍]          │ ← только active
│ ──────────────────────────────────────────────── │
│ Тип сечения      Прямоугольное 🔒                 │ ← fixed, копия авто
│ Ширина*          [ 500 ] [мм ▾]   (100…2000, ш.50)│ ← unit: число+единица
│ Высота*          [ 300 ] [мм ▾]                   │    + валидация min/max/step
│ Длина*           [ 1250 ] [мм ▾]                  │
│ Металл (корпуса)*[ оц.сталь ▾ ]                   │ ← enum: только выбор
│ Толщина (стенки) [       ] [мм ▾]                 │ ← необязательное
│ ──────────────────────────────────────────────── │
│ ИМЯ:     Воздуховод прямоугольный 500х300х1250    │ ← живой предпросмотр
│          оц.сталь                                 │    при каждом вводе
│ АРТИКУЛ: ВЗ-П-500-300-1250-ОЦ                     │
│                            [Отмена] [Создать]     │
└───────────────────────────────────────────────────┘

Форма ГЕНЕРИРУЕТСЯ из element_properties (порядок sort_order,
qualifier в подписи, 🔒 для fixed). Сборка имени/артикула — чистая
функция buildFromGroups(groups, values) НА ФРОНТЕ (и та же логика
на бэке при сохранении): группы → токены → пустая группа исчезает
с разделителем. Дубликат артикула → предупреждение «позиция уже есть».
```

3. ЭЛЕМЕНТЫ — конструктор шаблонов (технолог)
═══════════════════════════════════════════════════
```
✓ РЕШЕНИЕ 2026-07-21 (вместо мастер-детейла): свой уровень ВКЛАДОК —
[Список·pinned] + открытые элементы (тот же паттерн, что задачи в
разделе проекта). URL: /nomenclature/elements/{id|new}.

СПИСОК — таблица на всю ширину:
  │ Элемент │ Категория │ Статус (бейдж ТЕКСТОМ: активен/
  │ заблокирован/архив — не цветная точка) │ Позиций │
  клик по строке → элемент открывается ВКЛАДКОЙ;
  [+ Новый элемент] — вкладка «Новый элемент» (id=new).

ВКЛАДКА ЭЛЕМЕНТА — форма на всю ширину, деталь из 3 секций,
редактирование через ЧЕРНОВИК (механизм dirty 05/12 §6):
  изменения → dirty-точка ● на вкладке + бейдж «не сохранено»,
  кнопки [Сохранить][Отменить изменения] ТОЛЬКО при изменениях,
  SaveGuard: «Сохранить и закрыть» из диалога несохранённого.
  Сохранение нового элемента закрывает вкладку «Новый элемент»
  и открывает постоянную вкладку сохранённого.

┌ [Список] [Отвод прямоугольный ●] [Новый элемент] [+ Новый] ┐
│ ОТВОД ПРЯМОУГОЛЬНЫЙ #4 [активен] [не сохранено][Сохранить] │
│ ── 1. Основное ─ Категория [Вентиляция▾]                    │
│                  Тип изделия [Отвод▾] ← reference 57        │
│ ── 2. Свойства (element_properties) ──                      │
│ │Свойство│Тип изм.│Фикс.│Уточн.│Об.│⇅│                      │
│ │Тип сеч.│Тип сеч.│Прям🔒│ —  │ ✓ │ │                       │
│ │Ширина  │Длина   │  —  │ —   │ ✓ │ │  [+ Свойство]         │
│ ── 3. Шаблоны имени и артикула + предпросмотр ──            │
└─────────────────────────────────────────────────────────────┘

КОНСТРУКТОР ШАБЛОНОВ (группы токенов, 02/01 §2.7.1):
  группа = блок-чип; внутри — свойства + литералы (х, Ду, М — из
  справочника литералов); между группами авторазделитель
  (имя→пробел, артикул→дефис).
  ЭТАП 1 ✓: кнопочное управление (добавить группу, добавить токен
  в группу, ⇅ порядок, ✕) — компактно и надёжно.
  ЭТАП 2 ✓ (2026-07-21, по макету пользователя): DRAG-AND-DROP —
  палитра слева (свойства одноразовые / литералы и текст многоразовые,
  только внутрь группы / «Имя элемента»), холст с группами; тонкие
  полоски — зоны вставки; свойство между группами создаёт новую
  группу; токены и группы таскаются; текст редактируется кликом;
  зона удаления появляется снизу при перетаскивании.
  ПЕРЕКЛЮЧАТЕЛЬ режима [Перетаскивание|Кнопки] в секции шаблона
  (persist localStorage erp_tpl_editor) — обе версии живут.
  Артикул автопредзаполняется из свойств (каждое — своя группа).
  ПРЕДПРОСМОТР (2026-07-21, уточнено): КРУПНОЙ строкой НАД холстом.
  Разметка dnd-режима: слева палитра свойств, справа колонка из двух
  рядов — ряд 1 ПРЕДПРОСМОТР (ИМЯ / АРТИКУЛ моноширинным), ряд 2 холст;
  предпросмотр не заходит горизонтально под палитру. В кнопочном
  режиме — строкой над редактором. Меняешь — сразу видно; тестовые
  значения (500, 300…). Мелкий сводный блок внизу убран.
  ПОРЯДОК СВОЙСТВ (element_properties.sort_order, 02/01 §2.7):
  колонка ⇅ в таблице свойств элемента — стрелки вверх/вниз,
  перенумерация sort_order; влияет на форму позиции и подстановку
  тестовых значений предпросмотра. Тоже через черновик элемента.

IMMUTABLE (02/01 §2.7): у активного элемента С ПОЗИЦИЯМИ менять
  свойства/шаблоны нельзя — кнопка [Создать новую версию]:
  клонирует элемент → правки → сохранить → старый blocked +
  replaced_by, в списке «↳ см. новый». Пустой элемент (0 позиций)
  правится свободно.
```

```
[Свойства·pinned] [Единицы и значения] [Типы изделий] [Уточнения] [Категории] [Литералы]

> ✓ РЕШЕНИЯ 2026-07-21 (реализованы в ergo-erp): kind на типе;
> enum-тип → одно свойство; «Типы изделий» отдельной вкладкой;

СВОЙСТВА (properties + property_qualifiers):
  таблица: Название │ Тип измерения │ Допустимые уточнения
  уточнения per-свойство: показываются ТОЛЬКО выбранные (чипы с ✕)
  + селект «＋» добавления из qualifiers; «без уточнений» — норма
  (property_qualifiers защищает от ошибок — 02/01 §2.5)
  Валидация: на enum-тип измерения — максимум ОДНО свойство
  (02/01 §2.6); unit-типов не касается (Длина → Ширина/Высота/…)

УТОЧНЕНИЯ (qualifiers): простой CRUD-список (Наружный, Рабочее…)
  + колонка «используется в N свойствах»

ЕДИНИЦЫ И ЗНАЧЕНИЯ (reference_types + references) — мастер-детейл:
  слева типы: форма создания = имя + ВИД (перечисление|значение,
  02/01 §2.2 kind) — вид выбирается ОДИН раз на типе;
  Тип изделия (57) ★ поднят наверх списка; kind-бейдж у каждого типа
  справа значения типа (вид наследуется, у значения выбора НЕТ):
  Название │ Символ │ Код артикула (только enum-типы) │
  порядок ⇅ │ активно

ТИПЫ ИЗДЕЛИЙ — отдельная вкладка-настройка: CRUD значений типа 57
  (Название │ Код арт. │ активно). Те же данные, что тип 57 в
  «Единицах» — вынесены отдельно как ключевая настройка
  (первый фасет каталога + источник имён элементов)

КАТЕГОРИИ: плоский список + sort_order ⇅ (parent_id в запасе — 02/01)

ЛИТЕРАЛЫ (name_literals): CRUD символов (х, Ф, Ду , М, R=, EI…)
```
