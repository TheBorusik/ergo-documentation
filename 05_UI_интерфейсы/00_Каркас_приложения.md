# Каркас приложения (layout) — шапка, меню, контент, футер

Общая рамка всех экранов. Внутри неё рендерится контент
(дашборд, проекты, карточка и т.д.).

═══════════════════════════════════════════════════
СТРУКТУРА (layout)
═══════════════════════════════════════════════════
```
┌────────────────────────────────────────────────────┐
│ ШАПКА (header)                                      │
│ Лого · Поиск · Уведомления · Профиль               │
├──────────┬─────────────────────────────────────────┤
│          │                                         │
│ ЛЕВОЕ    │  КОНТЕНТ (меняется)                     │
│ МЕНЮ     │  вкладки / дашборд / карточка проекта    │
│ (nav)    │                                         │
│          │                                         │
├──────────┴─────────────────────────────────────────┤
│ ФУТЕР (footer) — версия · статус подключения       │
└────────────────────────────────────────────────────┘

Шапка и футер — на всю ширину.
Левое меню + контент — между ними.
```

═══════════════════════════════════════════════════
1. ШАПКА (header) — на всю ширину
═══════════════════════════════════════════════════
```
СЛЕВА:
  Лого + название (ВентЭРП)

ЦЕНТР:
  Глобальный поиск (проекты, позиции, задачи, поставщики)
  плейсхолдер: "Поиск проектов, позиций, задач…"

СПРАВА:
  Уведомления (колокольчик + индикатор непрочитанных)
  Профиль (аватар + имя + роль + выпадашка)
    выпадашка: профиль, настройки, выход

Высота ~48-52px. Фон белый, нижняя граница.
Присутствует на ВСЕХ экранах.
```

## Глобальный поиск (шапка)
```
Единое поле поиска по системе:
  проекты (по имени/коду)
  позиции (по номенклатуре)
  задачи
  поставщики/заказчики
Результаты сгруппированы по типу. Клик → открыть.
API: GET /api/search?q={query} → { projects[], positions[], tasks[], contractors[] }
```

## Уведомления (шапка)
```
Колокольчик + счётчик непрочитанных.
Клик → выпадающий список:
  назначена задача, пришло на согласование,
  оплата проведена, приближается срок...
Источник: события где пользователь адресат.
API: GET /api/notifications, POST /api/notifications/{id}/read
(можно этап 2 — на старте простой индикатор)
```

## Профиль (шапка)
```
Аватар (или инициалы) + имя + роль.
Выпадашка: Мой профиль, Настройки, Выход.
Роль показывается (Снабженец, Менеджер...) — понятно кто вошёл.
```

═══════════════════════════════════════════════════
2. ЛЕВОЕ МЕНЮ (nav) — вертикальное
═══════════════════════════════════════════════════
```
📊 Дашборд        — главная, настраиваемый дашборд
✅ Мои задачи     — задачи пользователя по роли
💼 Продажи        — воронка сделок, пресейл (05/14, под правом)
📁 Проекты        — все проекты (список + карточки)
─────────────
📦 Номенклатура   — конструктор изделий
🏭 Контрагенты    — заказчики/поставщики/подрядчики (05/13,
                    бывш. "Поставщики" — вопрос 6 в 05/13)
🛒 Закупки        — order по всем проектам (бухгалтер/рук.)
─────────────
📚 Справочники    — works, warehouses, users...
⚙️ Настройки      — админка (под правом)

Пункты под правами (<Can>): нет права → пункт скрыт.
Активный пункт подсвечен.
Ширина ~190-210px.
```

## Правила меню
```
- Без разделов-этапов в меню (не дробить на 6 этапов)
  "Мои задачи" заменяет их — вход в свою работу
- Разделы этапов живут ВНУТРИ проекта (карточка)
- Пункты под правами (Закупки — бухгалтер/рук., Настройки — админ)
- Активный пункт подсвечен accent
- Разделители группируют (работа / справочники / система)
```

═══════════════════════════════════════════════════
3. КОНТЕНТ (между меню и футером)
═══════════════════════════════════════════════════
```
Меняется по выбранному пункту меню:
  Дашборд → конструктор виджетов
  Проекты → лента вкладок (навигация табами — см. прототипы)
  Мои задачи → список по проектам
  и т.д.

Именно здесь работает система вкладок (гибрид:
верх плоский + проект рекурсивный, см. прототип).
```

═══════════════════════════════════════════════════
4. ФУТЕР (footer) — минимальный
═══════════════════════════════════════════════════
```
СЛЕВА:  ВентЭРП © 2026 (копирайт)
СПРАВА: статус подключения (● Подключено) + версия v1.0.0

Тонкий (~32px), неяркий.
Рабочее приложение — футер минимальный (не маркетинговый).
Можно вовсе убрать, оставив версию в углу меню.
Опционально: индикатор сохранения, статус синхронизации.
```

═══════════════════════════════════════════════════
5. АДАПТИВНОСТЬ (заметка)
═══════════════════════════════════════════════════
```
Десктоп — основной сценарий (рабочее место).
Меню сворачивается в иконки (узкий экран).
Шапка: поиск в иконку, профиль компактно.
Мобильный — этап 2 (если нужен на объекте — прораб).
```

═══════════════════════════════════════════════════
6. КОМПОНЕНТЫ КАРКАСА (для реализации)
═══════════════════════════════════════════════════
```
<AppShell>
  <Header>            — лого, поиск, уведомления, профиль
    <GlobalSearch />
    <Notifications />
    <UserMenu />
  <Sidebar>           — навигация (пункты под <Can>)
    <NavItem />
  <Content>           — роутинг сюда (Outlet)
  <Footer>            — версия, статус

Каркас постоянный, меняется только <Content>.
Пункты меню и элементы шапки — под правами.
```
