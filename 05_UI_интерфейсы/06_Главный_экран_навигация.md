# Главный экран и навигация (детально)

Структура левого меню, дашборд, вход в работу по ролям.

═══════════════════════════════════════════════════
ВОПРОС — разделы по этапам в меню?
═══════════════════════════════════════════════════
```
Идея: в левом меню разделы Проектирование/Снабжение/...
  чтобы инженер/снабженец заходил в свой и видел проекты.

Проблема: задачи привязаны к ПРОЕКТУ.
  "Снабжение" в меню → задачи всех проектов вперемешку,
  теряется контекст проекта.

Решение: раздел "МОИ ЗАДАЧИ" (по роли пользователя),
  сгруппированный по проектам. Даёт быстрый вход в свою
  работу БЕЗ дробления меню на 6 этапов.
```

═══════════════════════════════════════════════════
СТРУКТУРА ЛЕВОГО МЕНЮ (рекомендация)
═══════════════════════════════════════════════════
```
┌─────────────────┐
│ ЛОГО            │
├─────────────────┤
│ 📊 Дашборд      │  ← главный, обзор + метрики
│ ✅ Мои задачи   │  ← задачи пользователя по его роли
│ 📁 Проекты      │  ← все проекты (список + канбан)
│ ─────────────   │
│ 📦 Номенклатура │
│ 🏭 Поставщики   │
│ 📋 Справочники  │
│ ─────────────   │
│ ⚙️ Настройки    │  (админ)
└─────────────────┘

Без разделов-этапов в меню.
"Мои задачи" заменяет их — вход в свою работу.
Разделы этапов живут ВНУТРИ проекта (карточка).
```

═══════════════════════════════════════════════════
РАЗДЕЛ "МОИ ЗАДАЧИ" (вход в работу по роли)
═══════════════════════════════════════════════════
```
Показывает задачи ТЕКУЩЕГО пользователя (где он ответственный),
сгруппированные по проектам. Роль определяет какие задачи.

┌─ Мои задачи ─────────────────────────────────────┐
│ Фильтр: [активные][все]  [по проекту▾][по сроку]  │
├───────────────────────────────────────────────────┤
│ 📁 Завод сыра                                     │
│   Воздуховоды П1 → ПостА    🛒 закупка  срок 20.01│
│   Клапаны П1                ⏳ согласование        │
│ 📁 Офис BCG                                       │
│   Оборудование П2           🛒 закупка            │
├───────────────────────────────────────────────────┤
│ (снабженец видит задачи снабжения где он ответств.)│
└───────────────────────────────────────────────────┘

Снабженец → задачи снабжения (его)
Инженер → спецификации (его)
Прораб → работы (его)
Клик на задачу → деталь (внутри проекта)

→ быстрый вход: "что мне делать сейчас"
→ группировка по проектам (контекст сохранён)
```

═══════════════════════════════════════════════════
ГЛАВНЫЙ ЭКРАН — ДАШБОРД (детально)
═══════════════════════════════════════════════════
```
Первое что видишь при входе. Обзор + метрики.
Что показывать зависит от роли (права на виджеты).

┌─ Дашборд ────────────────────────────────────────────┐
│ ВЕРХНИЕ МЕТРИКИ (карточки):                           │
│ ┌────────┐┌────────┐┌────────┐┌────────┐             │
│ │Проектов││В работе││Срочных ││Просроч.│             │
│ │  12    ││   8    ││   3    ││   2    │             │
│ └────────┘└────────┘└────────┘└────────┘             │
├───────────────────────────────────────────────────────┤
│ ПРОЕКТЫ ПО ЭТАПАМ (распределение):                    │
│   Продажа 3 · Проектир. 2 · Снабжение 3 ·             │
│   Монтаж 2 · Докум. 1 · Сдача 1                       │
│   (мини-диаграмма или полоски)                        │
├───────────────────────────────────────────────────────┤
│ ФИНАНСЫ (если право ERP.Finance.View):                │
│   В работе: 45 млн · Ожидает оплаты: 8 млн            │
├─────────────────────────┬─────────────────────────────┤
│ МОИ ЗАДАЧИ (топ-5)       │ ПОСЛЕДНИЕ СОБЫТИЯ            │
│  - Согласовать заказ П1  │  - Доставлены воздуховоды   │
│  - Проверить спец. пом2  │  - Оплачен заказ Клапаны    │
│  [все мои задачи →]      │  - Завершена спец. пом1     │
└─────────────────────────┴─────────────────────────────┘

Виджеты дашборда (под правами):
  метрики проектов — все
  финансы — ERP.Finance.View (менеджер/бухгалтер)
  мои задачи — все (свои)
  события — из event_log (timeline последних)
```

═══════════════════════════════════════════════════
РАЗДЕЛ "ПРОЕКТЫ" (список + канбан)
═══════════════════════════════════════════════════
```
Все проекты. Переключатель вида.

┌─ Проекты                    [Список][Канбан]  [+ Новый]┐
│ Фильтр: [этап▾][менеджер▾][срок]  🔍                   │
├────────────────────────────────────────────────────────┤
│ СПИСОК (таблица):                                       │
│  Название    │Заказчик│Этап    │Прогресс│Срок │Отв.    │
│  Завод сыра  │Молоко  │Снабжен.│▓▓▓░ 60%│01.03│Иванов  │
│  Офис BCG    │BCG     │Монтаж  │▓▓▓▓ 80%│15.03│Петров  │
│  клик → карточка проекта                               │
└────────────────────────────────────────────────────────┘

КАНБАН (переключатель) — проекты по этапам:
  колонки = этапы, карточки = ПРОЕКТЫ
  (обзор где какой проект в целом)

Список — основной (детальный).
Канбан — обзорный (быстрый взгляд).
```

═══════════════════════════════════════════════════
ПОЧЕМУ НЕ РАЗДЕЛЫ-ЭТАПЫ В МЕНЮ
═══════════════════════════════════════════════════
```
Разделы Проектирование/Снабжение в ГЛАВНОМ меню — избыточно:

✗ задачи без контекста проекта (вперемешку все проекты)
✗ дублирует разделы ВНУТРИ проекта (там уже есть этапы)
✗ 6 пунктов меню вместо одного "Мои задачи"
✗ путаница: этап в меню vs этап в проекте

"Мои задачи" решает ту же потребность лучше:
✓ вход в свою работу (по роли)
✓ группировка по проектам (контекст)
✓ один пункт меню

Разделы-этапы уместны ВНУТРИ проекта (карточка),
там есть контекст конкретного проекта.
```

═══════════════════════════════════════════════════
ИТОГ — трёхуровневый доступ к работе
═══════════════════════════════════════════════════
```
1. ДАШБОРД — общая картина (метрики, распределение)
2. МОИ ЗАДАЧИ — моя работа (по роли, все проекты)
3. ПРОЕКТЫ → карточка → разделы этапов — работа в проекте

Пользователь:
  быстро глянул дашборд →
  пошёл в "Мои задачи" делать своё →
  ИЛИ открыл конкретный проект целиком

→ гибко: и по задачам (свои), и по проектам (целиком)
```

═══════════════════════════════════════════════════
API
═══════════════════════════════════════════════════
```
GET /api/dashboard
  → метрики (проектов, по этапам, финансы если право),
    мои задачи (топ), последние события (event_log)

GET /api/my-tasks?status=active
  → задачи текущего пользователя (где responsible_id=me)
    сгруппированы по проектам
  фильтр по роли неявный (его задачи его этапа)

GET /api/projects?view=list|kanban&filters
  → список/канбан проектов
```

═══════════════════════════════════════════════════
ВОПРОСЫ
═══════════════════════════════════════════════════
```
1. "Мои задачи" вместо разделов-этапов — согласен?
2. Дашборд — какие метрики главные для вас?
3. Финансы на дашборде — кому показывать?
4. Канбан проектов нужен или хватит списка?
5. "Мои задачи" — только свои или руководитель видит команду?
6. Быстрые действия с дашборда (создать проект)?
```
