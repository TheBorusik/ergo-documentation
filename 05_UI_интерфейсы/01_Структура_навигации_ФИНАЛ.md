# UI — финальная структура (master-detail по разделам)

Модель: карточка проекта → разделы (этапы) → список задач → деталь задачи.
Стандартный паттерн "список → открыл по id → детали".

═══════════════════════════════════════════════════
НАВИГАЦИЯ — ТРИ УРОВНЯ
═══════════════════════════════════════════════════
```
1. СПИСОК ПРОЕКТОВ (все проекты)
     ↓ клик на проект
2. КАРТОЧКА ПРОЕКТА → разделы (этапы) слева
     клик на раздел → список задач этапа
     ↓ клик на задачу
3. ДЕТАЛЬ ЗАДАЧИ (интерфейс свой для каждого этапа)

Хлебные крошки: Проекты › Завод сыра › Снабжение › Воздуховоды П1
```

═══════════════════════════════════════════════════
УРОВЕНЬ 1 — СПИСОК ПРОЕКТОВ
═══════════════════════════════════════════════════
```
Основной вид — таблица/список проектов:
  Название | Заказчик | Этап | Прогресс | Срок | Ответственный
  клик → карточка проекта
[+ Новый проект]
Фильтры: этап, менеджер, срок, поиск.
Отдельно: [Канбан] — доска проектов по этапам (обзорный вид).
```

═══════════════════════════════════════════════════
УРОВЕНЬ 2 — КАРТОЧКА ПРОЕКТА (разделы = этапы)
═══════════════════════════════════════════════════
```
┌─ Завод сыра ──────────────────────────────────────┐
│ Заказчик: Молоко  Этап: Снабжение  Срок: 01.03     │
├────────────┬───────────────────────────────────────┤
│ РАЗДЕЛЫ     │  Содержимое раздела                    │
│            │                                         │
│ Обзор      │  (список задач выбранного этапа         │
│ Проектир.  │   + кнопка создать задачу)              │
│▶Снабжение  │                                         │
│ Работы     │                                         │
│ Документы  │                                         │
│ ────────   │                                         │
│ Позиции    │  (особый раздел — все позиции)          │
│ КП         │                                         │
│ Файлы      │                                         │
└────────────┴───────────────────────────────────────┘

Разделы-этапы: Проектирование, Снабжение, Работы, Документы
  → каждый показывает список СВОИХ задач + [+ Задача]
Особые разделы:
  Обзор — сводка проекта, прогресс
  Позиции — все позиции со статусами (просмотр)
  КП — структура КП (группы/записи)
  Файлы — общие вложения
```

═══════════════════════════════════════════════════
РАЗДЕЛ ЭТАПА — список задач
═══════════════════════════════════════════════════
```
Клик на раздел (напр. Снабжение) → справа:

┌─ СНАБЖЕНИЕ                              [+ Задача] ─┐
│ ┌────────────────────────────────────────────────┐ │
│ │ Воздуховоды П1 → ПостА    🛒 в закупке  👤Петров│ │
│ │ Клапаны П1 → ПостБ        ✓ оплачено           │ │
│ │ Крепёж → ПостВ            📦 доставка           │ │
│ └────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────┘

Список задач этапа: название, статус, ответственный.
[+ Задача] → модалка создания (название) → задача в списке.
Клик на задачу → УРОВЕНЬ 3 (деталь).
```

═══════════════════════════════════════════════════
УРОВЕНЬ 3 — ДЕТАЛЬ ЗАДАЧИ (свой интерфейс этапа)
═══════════════════════════════════════════════════

## Общий каркас (у всех этапов)
```
┌─ [Название задачи] ──────────── [← назад] ────────┐
│ Статус: [...]                                      │
│ ПОЗИЦИИ задачи:            [+ выбрать/добавить]     │
│   ...                                              │
│ [СВОЯ РАБОТА ЭТАПА]                                │
│ ДОКУМЕНТЫ/файлы задачи:   [+ прикрепить]           │
│ [Завершить задачу]                                 │
└────────────────────────────────────────────────────┘
```

## Проектирование — деталь
```
Позиции: [+ добавить] (создать/выбрать номенклатуру)
  каждая позиция: привязка к КП [П1 ▾]
Чертежи: [+ прикрепить] (файлы проектирования)
[Завершить спецификацию] → позиции готовы к снабжению
```

## Снабжение — деталь
```
Позиции: [+ выбрать] (из доступных: готовые, не занятые)
Поставщик: [выбор]
Процесс (14 шагов): Запрос→Цены→Согласование→Оплата→Доставка
  каждый шаг — свои действия (модалки: отправка, маппинг цен)
[Завершить] → позиции доставлены
```

## Работы — деталь
```
Вид работ: [выбор из справочника]
Позиции: [+ выбрать] (из доставленных)
Документы работ: [+ прикрепить] (акты)
[Завершить] → позиции установлены
```

## Документация — деталь
```
Позиции: [+ выбрать] (из установленных)
  каждая: [выбрать документ ▾] (сертификат/паспорт)
[Завершить] → позиции задокументированы
```

═══════════════════════════════════════════════════
КЛЮЧЕВОЙ ПРИНЦИП
═══════════════════════════════════════════════════
```
Стандартный master-detail:
  раздел (этап) → список задач → клик → деталь по id

Каркас детали ОДИНАКОВ (позиции + действия + завершить),
интерфейс работы — СВОЙ у каждого этапа.

Создание задачи — в разделе этапа (ясно на что).
Документация — тоже задачи (единообразно со всеми).
```

═══════════════════════════════════════════════════
ОСОБЫЙ РАЗДЕЛ "ПОЗИЦИИ"
═══════════════════════════════════════════════════
```
Не про задачи — общий обзор ВСЕХ позиций проекта.
Статусы, группировка (КП/поставщик/этап), фильтры.
Клик на позицию → её история (весь путь по конвейеру).
Только просмотр/анализ.
```

═══════════════════════════════════════════════════
СВЯЗЬ РАЗДЕЛОВ (готовность позиций)
═══════════════════════════════════════════════════
```
Задача проектирования завершена → позиции "готовы к закупке"
  → в разделе Снабжение при [+ выбрать] они доступны

Индикатор готовности (на разделе):
  рядом с "Снабжение" — счётчик "5 позиций ждут"
  → снабженец видит что появилась работа

Позиции перетекают по этапам (конвейер),
задачи создаются в разделах по мере поступления.
```

═══════════════════════════════════════════════════
ОБЩИЙ КАНБАН ПРОЕКТОВ (отдельно, верхний уровень)
═══════════════════════════════════════════════════
```
На уровне СПИСКА проектов — переключатель [Список] [Канбан].
Канбан ПРОЕКТОВ: колонки = этапы, карточки = ПРОЕКТЫ.
Показывает где каждый ПРОЕКТ (общий статус).

Это ВНЕШНИЙ канбан (проекты по этапам).
Внутри проекта — НЕ канбан, а разделы+списки (master-detail).

Два уровня:
  внешний — проекты движутся по этапам (канбан/список)
  внутренний — разделы этапов со списками задач
```

═══════════════════════════════════════════════════
ВОПРОСЫ
═══════════════════════════════════════════════════
```
1. Разделы слева в карточке (этапы + Обзор/Позиции/КП/Файлы) — ок?
2. Деталь задачи — свой интерфейс, общий каркас — верно?
3. Индикатор готовности позиций на разделе — нужен? как?
4. Внешний канбан проектов (список проектов) — оставляем?
5. Позиции как отдельный раздел просмотра — ок?
6. Переход проекта между этапами — кнопка в шапке карточки?
```
