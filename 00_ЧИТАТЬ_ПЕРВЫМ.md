═══════════════════════════════════════════════════
КЛЮЧЕВЫЕ АРХИТЕКТУРНЫЕ РЕШЕНИЯ
═══════════════════════════════════════════════════
```
1. КОНВЕЙЕР: позиция — единый источник истины, течёт по этапам,
   меняя статус. Задачи не мигрируют (в своих этапах).
2. ЭКСКЛЮЗИВНОСТЬ: позиция в одной активной задаче за раз.
3. ДРОБЛЕНИЕ: разная судьба → отдельные позиции.
4. ЕДИНЫЙ МЕХАНИЗМ ПОДЗАДАЧ: получил→раздробил→передал (все этапы).
5. КП — скелет, этапы наполняют деталями.
6. ЕДИНОЕ ОКНО ВЫБОРА позиций (переиспользование).
7. МАППИНГ обучается, доказан на реальных счетах (89%).
8. СТАТУСЫ ЭТАПОВ: ожидание/в работе/завершён (наглядность на доске).
9. УНИФИЦИРОВАННАЯ ИСТОРИЯ: все события в один журнал event_log.
10. УВЕДОМЛЕНИЯ: notifications в БД (адресно, is_read) ≠ event_log
    (аудит для всех) ≠ push (сигнал "обновись", будит счётчик).
```
