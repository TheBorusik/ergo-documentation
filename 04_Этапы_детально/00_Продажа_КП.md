# ЭТАП 0 — ПРОДАЖА (создание КП) — полная детализация

Формат: интерфейс (ASCII) → действия → API (ссылки на МОДЕЛИ_API.md) → таблицы.
Первый этап. Отличается от 1-5: не конвейер позиций, а построение КП.

═══════════════════════════════════════════════════
1. НАЗНАЧЕНИЕ
═══════════════════════════════════════════════════
```
Менеджер создаёт проект и коммерческое предложение (КП):
блоки (группы) + записи (строки). Блоки по умолчанию —
базовые. Записи вводятся вручную с автодополнением по истории.

Роль:  Менеджер
Вход:  новый проект (тендер/прямой заказ)
Выход: КП сформировано, проект выигран → Проектирование
```

## 2.1 Создание проекта (модалка)
```
┌─ Новый проект ─────────────────────────────┐
│ Название: [Завод сыра — вентиляция]        │
│ Заказчик: [выбор ▾] или [+ новый]          │

└──────────────────────────────────────────────┘
→ проект создан (status=active, этап Продажа)
→ КП с базовыми блоками автоматически
```

## 2.2 Раздел "КП" (построение)
```
┌──────────────────────────────────────────────────────────────┐
│ КП — Завод сыра                    [+ Блок] [Импорт xlsx]    │
├──────────────────────────────────────────────────────────────┤
│ ▾ Оборудование                                    [⋯]        │
│   ┌────────────────────────────────────────────────────────┐ │
│   │ Приточная установка П1          1  шт          [✕]     │ │
│   │ Приточная установка П2          1  шт          [✕]     │ │
│   │ [+ добавить строку...]                                  │ │
│   └────────────────────────────────────────────────────────┘ │
│ ▾ Холодоснабжение                                 [⋯]        │
│   ┌────────────────────────────────────────────────────────┐ │
│   │ Наружный блок MDVO-VCM560       2  шт          [✕]     │ │
│   │ [+ добавить строку...]                                  │ │
│   └────────────────────────────────────────────────────────┘ │
│ ▾ Автоматизация                                   [⋯]        │
│ ▾ Теплоснабжение                                  [⋯]        │
│ ▾ Материалы                                       [⋯]        │
│ ▾ Работы                                          [⋯]        │
├──────────────────────────────────────────────────────────────┤
│ Всего записей: 29                                            │
└──────────────────────────────────────────────────────────────┘

Внутри блока — строки (записи), ввод вручную.
```

## 2.3 Ввод строки с автодополнением
```
┌─ Строка КП (ввод) ──────────────────────────────┐
│ Наименование: [Приточная уст___]                 │
│   ┌──────────────────────────────────────────┐   │
│   │ Приточная установка П1        (исп. 15)   │   │ ← подсказки
│   │ Приточная установка П2        (исп. 12)   │   │   из истории
│   │ Приточная установка (без охл.) (исп. 8)   │   │
│   └──────────────────────────────────────────┘   │
│ Кол-во: [1]    Ед: [шт ▾]                        │
│         [Добавить]                               │
└────────────────────────────────────────────────────┘

Автодополнение из истории ранее введённых записей.
(исп. N) — частота использования, популярные выше.
Можно ввести новое (добавится в историю).
```

│ (или выбрать типовой ▾)            │
│         [Отмена] [Добавить]        │
└──────────────────────────────────────┘

═══════════════════════════════════════════════════
4. ПРАВИЛА
═══════════════════════════════════════════════════
```
- Базовые блоки создаются автоматически при новом КП:
  Оборудование, Холодоснабжение, Автоматизация,
  Теплоснабжение, Материалы, Работы
- Можно добавлять свои блоки
- Строки вводятся вручную с автодополнением
- Автодополнение — из истории (уникальные записи, частота)
- Новая запись → сохраняется в историю (обучение)
- Цены — пока заглушка (простой вариант без цен)
- Раскрытие в позиции — на этапе Проектирования (не здесь)
- Выигрыш → status=won, переход в Проектирование
- Проигрыш → status=lost (проект в системе, архив)
```

═══════════════════════════════════════════════════
5. АВТОДОПОЛНЕНИЕ (механизм)
═══════════════════════════════════════════════════
```
Хранилище уникальных записей (накопитель):
  kp_item_suggestions
    name (уникальное название)
    block_name (в каком блоке обычно)
    times_used (частота)
    last_used_at

При вводе строки:
  поиск похожих по name → выпадающие подсказки
  сортировка по times_used (популярные выше)

При добавлении строки:
  если name новое → добавить в suggestions (times_used=1)
  если существует → times_used++

→ система учится на введённых записях, ускоряет ввод
  (простой аналог маппинга — обучение на истории)
```

6. API (детально)
═══════════════════════════════════════════════════

## 6.1 Создание проекта
```
POST /api/projects
  Тело: { name: string, customer_id: int, type: enum }
  Ответ: 201 Project  (status=active, этап Продажа)
  Побочно: создание базовых kp_groups (по шаблону)

GET /api/kp/default-blocks
  (шаблон базовых блоков)
  Ответ: 200 { blocks: string[] }
    ["Оборудование","Холодоснабжение","Автоматизация",
     "Теплоснабжение","Материалы","Работы"]
```

## 6.2 Блоки (группы КП)
```
GET /api/projects/{projectId}/kp
  (всё КП: блоки + записи)
  Ответ: 200 { groups: [ KpGroup + { items: KpItem[] } ] }

POST /api/projects/{projectId}/kp/groups
  Тело: { name: string }
  Ответ: 201 KpGroup

PATCH /api/kp/groups/{id}
  Тело: { name?: string, order?: int }
  Ответ: 200 KpGroup

DELETE /api/kp/groups/{id}
  Ответ: 204  (только если пустой или с подтверждением)
```

## 6.3 Записи (строки КП)
```
POST /api/kp/groups/{groupId}/items
  Тело: { name: string, quantity: decimal, unit: string }
  Ответ: 201 KpItem
  Побочно: обновить kp_item_suggestions (обучение)

PATCH /api/kp/items/{id}
  Тело: { name?, quantity?, unit? }
  Ответ: 200 KpItem

DELETE /api/kp/items/{id}
  Ответ: 204
```

## 6.4 Автодополнение
```
GET /api/kp/suggestions?q={query}&limit=10
  (подсказки записей из истории)
  Ответ: 200
    { items: [ { name: string, block_name: string,
                 times_used: int } ] }
    сортировка по times_used desc
```

## 6.5 Импорт и выигрыш
```
POST /api/projects/{projectId}/kp/import
  Тело: multipart (xlsx)
  Ответ: 200 { groups: [...] }  (распарсенное КП)

POST /api/projects/{projectId}/win
  Ответ: 200 Project  (status → won, этап → Проектирование)

POST /api/projects/{projectId}/lose
  Ответ: 200 Project  (status → lost)
```

───────────────────────────────────────────────────
POST /api/projects
Создать проект (+ базовые блоки КП авто)
───────────────────────────────────────────────────
```

3. project_stages → INSERT × 6 (по одной на этап:

                    → 6 этапов проекта со статусами

4. kp_groups      → INSERT базовые блоки (batch):
                    Оборудование, Холодоснабжение, Автоматизация,
                    Теплоснабжение, Материалы, Работы
                    → базовые группы КП по умолчанию

5. event_log      → INSERT 'project_created'

Возврат: Project

[SQL] Шаги 3+4 — batch INSERT (6 этапов, 6 блоков).

PATCH /api/kp/groups/{id} + DELETE
Переименовать / удалить блок
───────────────────────────────────────────────────
```
PATCH:
  1. kp_groups    → UPDATE name / order где id={id}
  2. event_log    → INSERT 'kp_block_renamed'

DELETE:
  1. kp_items     → SELECT COUNT где group_id={id}
                    → проверка: пустой? (или подтверждение)
  2. kp_items     → DELETE где group_id={id} (если с подтверждением)
  3. kp_groups    → DELETE где id={id}
  4. event_log    → INSERT 'kp_block_deleted'
```

───────────────────────────────────────────────────
POST /api/kp/groups/{groupId}/items
Добавить запись (строку) + обучение автодополнения
───────────────────────────────────────────────────
```
Тело: { name, quantity, unit }

1. kp_groups      → SELECT где id={groupId}  → проверка + название блока

2. kp_items       → SELECT MAX(order) где group_id={groupId}
                    → порядок

3. kp_items       → INSERT (group_id, name, quantity, unit,
                    source='kp', order=max+1)

4. kp_item_suggestions → SELECT где name={name}
                    → есть ли уже такая запись в истории

5. kp_item_suggestions → UPDATE times_used++, last_used_at
                    (если есть) ИЛИ
                    INSERT (name, block_name, times_used=1)
                    (если новая)
                    → ОБУЧЕНИЕ автодополнения

6. event_log      → INSERT 'kp_item_added'

Возврат: KpItem

[Обучение: каждая запись пополняет suggestions для автодополнения]
