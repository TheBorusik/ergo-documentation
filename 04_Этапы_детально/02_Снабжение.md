### Ручной выбор при "не найдено" — выпадающий список
```
Строка прайса не смаплена → [Выбрать позицию задачи ▾]:

┌─ Выбрать позицию задачи ────────────┐
│ Клапан 400х400 ×2                    │  ← НЕПРИВЯЗАННЫЕ
│ Хомут крепёжный 500 ×20              │    позиции ЭТОЙ задачи
│ Переход 500/400 ×2                   │    (не всей базы)
│ ─────────────────                    │
│ ⊘ Игнорировать строку               │  ← доставка/услуга/лишнее
│   (доставка/услуга)                  │
└──────────────────────────────────────┘

ЛОГИКА:
  список = позиции задачи БЕЗ маппинга (непривязанные)
  по мере маппинга список сокращается (смапленные уходят)
  → мапим строки прайса на позиции задачи, не на всю номенклатуру

СТРОКИ-НЕ-ПОЗИЦИИ (доставка, услуга):
  "Доставка до объекта, 1 рейс" — не наша номенклатура
  → [Игнорировать строку] → не мапится
  → цена может учитываться отдельно (в стоимость заказа) или игнор

ПОЧЕМУ позиции задачи, а не вся база:
  мы отправили запрос с позициями задачи →
  поставщик ответил на них → строки мапятся на них
  (короткий список, логичный контекст)

НЮАНС (отложено): поставщик разбил нашу позицию на несколько строк
  (просили "Комплект" → прислал 3 составляющие)
  → несколько строк на одну позицию — усложнение, позже

│ Готовность у поставщика: [20.01]           │
│ Склад доставки: [Склад проекта ▾]          │
│   ○ Склад проекта "Завод сыра"             │
│   ○ Общий склад                            │
│ ───────────────────────────────           │
│ ДОКУМЕНТЫ (приехали с доставкой):          │
│   📄 сертификат_воздуховоды.pdf   [✕]      │
│   📄 паспорт_вентилятор.pdf       [✕]      │
│   [+ Прикрепить документ]                  │
│   (сертификаты/паспорта → в справочник)    │
│ ───────────────────────────────           │
│         [Отметить доставленным]            │
└──────────────────────────────────────────────┘
→ позиции delivered, документы в справочник, задача done

ДОКУМЕНТЫ С ДОСТАВКОЙ:
  документы приезжают ВМЕСТЕ с товаром →
  снабженец прикрепляет при доставке →
  попадают в справочник documents →
  позже (этап документации) навешиваются на позиции

Обязательность: НЕ обязательно для закрытия.
  документов может не быть (приедут позже, запрос отдельно) →
  задача закрывается без блокировки
  прикрепление — опционально при доставке

Уровень привязки: просто ЗАГРУЗКА в справочник.
  снабженец НЕ думает какой документ к чему →
  просто прикладывает файлы что приехали → documents
  привязка к позициям — работа этапа документации
```
