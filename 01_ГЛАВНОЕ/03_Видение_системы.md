# Видение системы — ERP для инжиниринговой фирмы
## Вентиляция и холодильное оборудование "под ключ"

---

## Что за бизнес

Инжиниринговая фирма полного цикла:
```
Проектирование → Поставка → Установка → Настройка
```
Работа ведётся ПРОЕКТАМИ. Каждый проект проходит путь от тендера до сдачи.

---

## Главная метафора — КАНБАН проектов

Проект движется по статусам слева направо. Доска канбан — центральный экран системы.

```
┌──────────┬──────────┬──────────┬──────────┬──────────┬──────────┐
│ ТЕНДЕР   │ ДОГОВОР  │ ПРОЕКТИР.│ СНАБЖЕНИЕ │  РАБОТЫ  │  СДАЧА   │
├──────────┼──────────┼──────────┼──────────┼──────────┼──────────┤
│ Завод    │ ТЦ Мега  │ Склад    │ Офис BCG │ Ресторан │ Гостиница│
│ сыра     │          │ Лента    │          │ Пушкин   │ Хаятт    │
│          │ Бизнес-  │          │          │          │          │
│ Аптека   │ центр    │ Цех №3   │          │          │          │
└──────────┴──────────┴──────────┴──────────┴──────────┴──────────┘
   карточка = проект, движется по статусам
```

---

## Стадии проекта (предварительно)

| # | Стадия | Что происходит | Ключевые сущности |
|---|---|---|---|
| 1 | **Тендер** | Участие в тендере, оценка, выигрыш | Проект (черновик), смета предв. |
| 2 | **Договор** | Заключение договора с заказчиком | Договор, сроки, бюджет |
| 3 | **Проектирование** | Разработка проекта, спецификация номенклатуры | Спецификация (наши позиции + кол-во) |
| 4 | **Снабжение** | Выбор поставщиков, заказы, приёмка на склад | Маппинг, заказы, склад |
| 5 | **Работы** | Монтаж, установка на объекте | Бригады, графики, акты |
| 6 | **Сдача** | Пусконаладка, настройка, сдача заказчику | Акты сдачи, документация |

*Статусы предварительные — уточняются.*

## Слои системы

```
┌─────────────────────────────────────────────────────────┐
│  УПРАВЛЕНИЕ ПРОЕКТАМИ (канбан, статусы, движение)        │  ← верхний слой
├─────────────────────────────────────────────────────────┤
│  Тендеры │ Договоры │ Проектир. │ Снабжение │ Работы │ Сдача │  ← стадии
├─────────────────────────────────────────────────────────┤
│  ПОДДЕРЖИВАЮЩИЕ МОДУЛИ:                                   │
│  • Справочник номенклатуры (эталон)      ✓ готов         │
│  • Поставщики и маппинг                  ~ частично      │
│  • Склад                                 нужно           │
│  • Контрагенты (заказчики, поставщики)   нужно           │
│  • Документы (договоры, акты)            нужно           │
│  • Люди/бригады                          нужно           │
└─────────────────────────────────────────────────────────┘
```

Справочник номенклатуры — это **фундамент**, который используется на стадии проектирования и снабжения. Но вся система шире.

---

## Что меняется в подходе

```
Было (моё понимание):        Стало (реальность):
─────────────────────        ──────────────────────
Справочник номенклатуры      ERP для инжиниринга
+ снабжение                  Номенклатура = один из модулей
                             Ядро = управление проектами (канбан)
                             Полный цикл: тендер → сдача
```

---

## Архитектурный принцип

```
ПРОЕКТ — центральная сущность, всё крутится вокруг него
   │
   ├─ имеет СТАТУС (стадию канбана)
   ├─ имеет ЗАКАЗЧИКА (контрагент)
   ├─ имеет ДОГОВОР
   ├─ имеет СПЕЦИФИКАЦИЮ (наши позиции номенклатуры)
   ├─ имеет ЗАКАЗЫ поставщикам (снабжение)
   ├─ имеет СКЛАД (свой или общий)
   ├─ имеет РАБОТЫ (монтаж)
   └─ имеет ДОКУМЕНТЫ (договоры, акты, сдача)
```

## Карта модулей (обновлённая)

```
ЯДРО
  projects                  — проекты + статус (канбан)
  project_statuses          — справочник стадий

СПРАВОЧНИКИ (master data)
  Номенклатура (12 таблиц)  ✓ готов
  contractors               — контрагенты (заказчики + поставщики)
  warehouses                — склады

ПРОЕКТИРОВАНИЕ
  project_positions         — спецификация (наши позиции + кол-во)

СНАБЖЕНИЕ
  supplier_mappings         ✓ частично
  supplier_offers           — предложения поставщиков
  orders, order_items       — заказы
  receipts, receipt_items   — приёмка (частичная)

РАБОТЫ (позже)
  work_packages             — пакеты работ
  teams                     — бригады
  schedules                 — графики

ДОКУМЕНТЫ (позже)
  contracts                 — договоры
  acts                      — акты

СИСТЕМНОЕ (позже)
  users, roles              — пользователи и права
  audit_log                 — аудит
```

## Приоритеты реализации (от ценного к вспомогательному)

```
Фаза 1 — Фундамент
  1. Справочник номенклатуры (готов)
  2. Проекты + канбан (статусы, движение)
  3. Контрагенты

Фаза 2 — Основной цикл
  4. Спецификация проекта (проектирование)
  5. Маппинг поставщиков
  6. Снабжение (выбор, заказы)
  7. Склад и приёмка

Фаза 3 — Полный цикл
  8. Договоры и документы
  9. Работы и бригады
  10. Сдача

Фаза 4 — Обвязка
  11. Права доступа
  12. Аудит, отчёты, аналитика
```
