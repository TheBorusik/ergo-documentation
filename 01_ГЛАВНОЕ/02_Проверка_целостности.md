# Проверка целостности — все этапы (сквозная)

Проверка что этапы стыкуются, статусы передаются, ничего не забыто.

═══════════════════════════════════════════════════
1. КОНВЕЙЕР СТАТУСОВ ПОЗИЦИИ (сквозной)
═══════════════════════════════════════════════════
```
ЭТАП 0 Продажа:      (позиций нет, только КП)
ЭТАП 1 Проектирование: created → spec_ready       ✓
ЭТАП 2 Снабжение:    spec_ready → in_purchase → purchased → delivered ✓
ЭТАП 3 Работы:       delivered → installed        ✓
ЭТАП 4 Документация: installed → documented        ✓
ЭТАП 5 Сдача:        (проверка documented) → проект completed ✓

ВХОД каждого этапа = ВЫХОД предыдущего:
  Снабжение вход spec_ready = выход Проектирования ✓
  Работы вход delivered = выход Снабжения ✓
  Документация вход installed = выход Работ ✓
  Сдача проверяет documented = выход Документации ✓

СТЫКОВКА ПОЛНАЯ — разрывов нет.
```

═══════════════════════════════════════════════════
2. ЕДИНОЕ ОКНО ВЫБОРА (переиспользование)
═══════════════════════════════════════════════════
```
Используется на этапах 2,3,4 с параметром stage:
  Снабжение:    selectable = spec_ready + свободна ✓
  Работы:       selectable = delivered + свободна ✓
  Документация: selectable = installed + свободна ✓

Один компонент, один API (positions/available?stage=).
Группировка spec|kp везде. Дробление, цвет, merge. ✓
```

═══════════════════════════════════════════════════
3. ЭКСКЛЮЗИВНОСТЬ (current_subtask_id)
═══════════════════════════════════════════════════
```
Проверка: позиция в одной активной задаче.
  Выбор в задачу → current_subtask_id = задача ✓
  Завершение задачи → current_subtask_id = null (освобождение) ✓
  Следующий этап видит свободную позицию ✓

Согласовано на всех этапах 2,3,4. ✓
```

═══════════════════════════════════════════════════
10. ЧТО ОТЛОЖЕНО (осознанно, не забыто)
═══════════════════════════════════════════════════
```
ЦЕНЫ (вся подсистема):
  - cost/sales/маржа в КП
  - плановая/фактическая
  - финальная оплата заказчика
  - множитель "1.00"

СТЫКУЕМОСТЬ/КОМПЛЕКТНОСТЬ (завтра уточнить):
  - element_relations (совместимость, требует)
  - подсказки при заказе несовместимого

ОТДЕЛЬНЫЕ МЕХАНИЗМЫ:
  - запрос отсутствующих документов у поставщика
  - частичная доставка
  - перезапись истёкших документов
  - шаблон паспорта (генерация)
  - объём работы (есть ли понятие)

НЕ СПРОЕКТИРОВАНО:
  - Справочники детально (contractors, users, warehouses формы)
  - Модуль document_templates (детально)
  - Права доступа по ролям (кто что видит/может)
  - Аудит действий
  - Дашборд (метрики) — концепция есть
  - Доска-канбан задач (обзорная) — концепция есть
  - Список проектов (главный экран) — концепция есть
  - Ускорители проектирования (наборы, копирование)
```

═══════════════════════════════════════════════════

═══════════════════════════════════════════════════
```
- "Взять в работу" — обязательный шаг или сразу

- Согласование построчно или вся задача
- Откат при отклонении согласования

- Автодополнение общее или по типам
- Раздел над группами КП (был в реальном КП)
- Отдельная таблица handover или поля в projects
```
