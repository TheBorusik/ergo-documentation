# Архитектура приложения — разбивка на модули
## На основе наброска процесса (6 этапов)

---

## ПРОЦЕСС ЦЕЛИКОМ (из наброска)

```
1. Продажа        → менеджер, КП/тендер, договор, укрупнённый список+цены
2. Проектирование → инженер, чертежи, реальные номенклатуры, части к закупке
3. Снабжение      → снабженец, выбор поставщика, склад, заказы, остатки
4. Монтаж и ПНР   → прораб, задачи на работы, отслеживание, закрытие
5. Документация   → сотрудник, паспорта установок, документы проекта
6. Сдача          → приёмка заказчиком, статус "завершён"
```

---

## КЛЮЧЕВОЕ НАБЛЮДЕНИЕ — два уровня детализации

Важный момент из наброска: проект описывается на ДВУХ уровнях.

```
УРОВЕНЬ 1 — Укрупнённый (этап продажи)
  "Приточная установка П1 — 495 636 ₽"
  "Воздухораспределительная сеть — 4 353 531 ₽"
  → крупные блоки + цена + количество (как КП)
  → это СИСТЕМЫ/УЗЛЫ проекта, не детальные позиции

УРОВЕНЬ 2 — Детальный (этап проектирования)
  Под каждую установку → реальные номенклатуры
  "Воздуховод 500х300, Отвод, Клапан..." с количествами
  → наши эталонные позиции из справочника
```

Связь: укрупнённый блок «Приточная установка П1» на этапе проектирования
раскрывается в список конкретных номенклатур.

## МОДУЛИ СИСТЕМЫ

### Ядро
```
МОДУЛЬ 0 — ПРОЕКТЫ (центральная сущность)
  Проект, его статус/этап, движение по этапам
  Заказчик, договор, общая стоимость
```

### Справочники (master data)
```
МОДУЛЬ S1 — НОМЕНКЛАТУРА (эталон)  ✓ готов (13 таблиц)
  Элементы, свойства, позиции, генерация имён/артикулов

МОДУЛЬ S2 — КОНТРАГЕНТЫ
  Заказчики, поставщики

МОДУЛЬ S3 — СКЛАДЫ
  Виртуальные склады: проектные + общие

МОДУЛЬ S4 — РАБОТЫ (справочник видов работ)
  Типы работ, расценки (для КП и задач)
```

### Этапы процесса (функциональные модули)
```
МОДУЛЬ 1 — ПРОДАЖА
  КП (укрупнённый список систем + работы + цены)
  Тендер/договор, выигрыш → старт проекта

МОДУЛЬ 2 — ПРОЕКТИРОВАНИЕ
  Детализация: укрупнённые блоки → реальные номенклатуры
  Чертежи (файлы), спецификация
  Части готовые к закупке (параллельно)

МОДУЛЬ 3 — СНАБЖЕНИЕ
  Маппинг поставщиков, выбор поставщика по позициям
  Выбор склада доставки, заказы (частями)
  Остатки/запас, статусы закупки

МОДУЛЬ 4 — МОНТАЖ И ПНР
  Задачи на работы, отслеживание, закрытие
  Привязка к поставленным позициям

МОДУЛЬ 5 — ДОКУМЕНТАЦИЯ
  Паспорта установок, документы проекта
  Генерация/хранение для передачи заказчику

МОДУЛЬ 6 — СДАЧА
  Приёмка заказчиком, акты, статус "завершён"
```

### Системное (обвязка)
```
МОДУЛЬ X1 — ПОЛЬЗОВАТЕЛИ И РОЛИ
  Менеджер, инженер, снабженец, прораб, сотрудник
  Права по этапам

МОДУЛЬ X2 — ФАЙЛЫ И ДОКУМЕНТЫ
  Чертежи, договоры, паспорта, акты (хранилище)
```

## РОЛИ (из наброска)

| Роль | Этап | Что делает |
|---|---|---|
| Менеджер | Продажа | КП, тендер, договор |
| Инженер | Проектирование | Чертежи, детализация номенклатур |
| Снабженец | Снабжение | Выбор поставщика, склад, заказы |
| Прораб | Монтаж/ПНР | Задачи на работы, контроль |
| Сотрудник | Документация | Паспорта, документы |
| (Заказчик) | Сдача | Приёмка |

## ВАЖНЫЕ ОСОБЕННОСТИ (из наброска)

### 1. Параллельность (не строго последовательно)
```
"не обязательно ждать пока проект будет готов, можно закупать частями"

→ Проектирование и Снабжение ПЕРЕСЕКАЮТСЯ
→ Часть спроектирована → уже в закупку
→ Закуплено частично → уже в монтаж
```

### 2. Части готовые к закупке
```
На этапе проектирования появляются ГОТОВЫЕ ЧАСТИ
→ помечаются как "готово к закупке"
→ уходят в снабжение не дожидаясь всего проекта
(это наши "наборы" из прошлого обсуждения)
```

### 3. Виртуальные склады
```
Склад проектный (свой под проект) ИЛИ общий
Выбор склада при закупке
Остатки/запас → перемещение на склад после реализации
```

### 4. Запас и остатки
```
Заказывают с запасом
После монтажа остатки → перемещать на (общий?) склад
→ нужен учёт движения: заказано / использовано / остаток
```

## ОТКРЫТЫЕ ВОПРОСЫ ПО НАБРОСКУ

1. **Укрупнённый список (КП)** — это справочник "систем" (Приточная установка П1)?
   Или свободный текст с ценами? Как он связан с детальными номенклатурами?

2. **Связь КП → номенклатура** — "Приточная установка П1" в КП = набор
   реальных позиций после проектирования. Как моделировать эту связь?

3. **Работы** — в КП есть строка "Работы 8 109 252 ₽" и "ПНР".
   Работы это отдельные позиции со своими расценками? Справочник работ?

4. **Цены** — в КП цены укрупнённые. Цены закупки (у поставщика) отдельные.
   Как соотносятся плановая (КП) и фактическая (закупка) стоимость?

5. **Части к закупке** — это наши "наборы"? Кто и как их формирует
   на этапе проектирования?

6. **Остатки** — после монтажа остаток на склад. Это движение между
   складами? Учёт "сколько заказано / ушло в монтаж / осталось"?

7. **Статусы проекта** — этап = статус всего проекта? Или этапы могут
   идти параллельно (часть в снабжении, часть в монтаже)?

8. **Паспорта/документы** — генерируются системой или загружаются файлами?

## УТОЧНЕНИЯ (раунд 1)

### Связь "раскрытие" КП → детали
```
КП-строка (укрупнённая) раскрывается в детальные сущности
через отдельную таблицу связи.

Приточная установка П1 ──раскрытие──► номенклатуры (позиции + кол-во)
Работы                 ──раскрытие──► конкретные работы (виды + объём)

Черновик:
  project_kp_items     — строки КП (название, цена заказчику, кол-во)
  kp_breakdown         — связь КП-строки с деталями:
                         kp_item_id → position_id + qty (номенклатура)
                         kp_item_id → work_id + qty (работа)

→ детальная проработка позже
```

### Три уровня цен
```
1. Цена ЗАКАЗЧИКУ   — в КП, видит клиент (с маржой)
                      хранится на project_kp_items
2. Цена ПЛАНОВАЯ    — расчётная на закупку (бюджет)
                      на позиции спецификации
3. Цена ФАКТИЧЕСКАЯ — реально потрачено после закупки
                      на order_item (материалы) + работах

Внутреннее отображение: план vs факт vs цена заказчика
→ маржа, отклонения, контроль бюджета

→ точная модель цен уточняется (пока набросок)
```

### Статус уточнений
```
КП детально       — позже (ждём реальный пример)
Работы            — позже (справочник + раскрытие)
Цены              — набросок есть, детали позже
Связь "раскрытие" — принцип зафиксирован (таблица связи)
```
