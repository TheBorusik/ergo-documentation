# Структура реального КП (разбор примера)

---

## Иерархия — ТРИ уровня

```
РАЗДЕЛ
  Оборудование и материалы раздел ВиК
  (+ Total cost / Total sales / маржа %)
    │
    ГРУППА
      Оборудование
      Холодоснабжение П2
      Холодоснабжение П3
      Автоматизация
      Кондиционер для отделения декорации
      Теплоснабжение приточных установок
      Материалы
        │
        СТРОКА (позиция КП)
          name, количество, цена/ед, валюта, маржа %, множитель
```

---

## Примеры строк (разная природа)

```
СЛОЖНЫЕ (раскроются в много номенклатур):
  Приточная установка П1 (без охлаждения) ВЕЗА   ×1   309 083 ₽
  Приточная установка П2 (с охлаждением)         ×1   658 333 ₽
  Вытяжная установка МО2                         ×1    58 083 ₽

ГОТОВЫЕ ИЗДЕЛИЯ (конкретный продукт):
  Наружный блок MDVO-VCM560V2R1A MDV   ×2   398 000 ₽
  Шкаф управления П1                   ×1    60 000 ₽
  Сплит-система Hisense 3,5 кВт        ×1    30 000 ₽

МАТЕРИАЛЫ (метраж/количество):
  Воздуховоды оц      ×720 (м)   1 100 ₽
  Фасонка оц.         ×144        1 800 ₽
  Теплоизоляция 10мм  ×400        800 ₽
  Трасса межблочная   ×30 (м)    2 000 ₽

КОМПЛЕКТЫ (набор):
  Комплект арматуры для смесительных узлов + трубы  ×1  444 166 ₽
  Комплект частотных преобразователей П1,П2,П3       ×3   18 000 ₽
  Решетки внутренние и наружные, комплект           ×1  325 000 ₽
```

## Поля строки КП (из примера)

```
- Наименование    "Приточная установка П1 (без охлаждения) ВЕЗА"
- Количество      1, 2, 30, 720...
- Цена за единицу  309 083 ₽ (это COST — себестоимость)
- Валюта          RUB
- Маржа %         40% (у некоторых строк, иначе общая по разделу)
- Множитель       1.00 (коэффициент? уточнить)
```

---

## Две цены уже в КП

```
Total cost:  7 310 416 ₽  (себестоимость — внутренняя)
Total sales: 10 223 134 ₽  (продажа заказчику)
Маржа: 57% (общая), у строк может быть своя (40%)

→ цена/ед в строках = COST (себестоимость)
→ sales = cost + маржа
→ две цены в одной строке: cost и sales (через маржу)
```

## Связь с моделью проекта

```
РАЗДЕЛ КП    → проект (или раздел проекта)
ГРУППА КП    → группа блоков (block_group?)
СТРОКА КП    → корневая подзадача (блок), раскрывается в номенклатуры

Иерархия subtasks:
  Группа "Оборудование" (parent=null, тип=группа)
    └─ "Приточная установка П1" (parent=Оборудование, тип=блок)
         └─ наполняется номенклатурами при проектировании
              Воздуховод ×14, Клапан ×2...
```

## Уточнение модели под реальный КП

```
Иерархия в subtasks (или отдельные таблицы):

УРОВЕНЬ 1 — Группа КП
  Оборудование, Холодоснабжение, Автоматизация, Материалы...
  (можно добавлять новые)

УРОВЕНЬ 2 — Строка КП (блок)
  Приточная П1, Воздуховоды оц, Наружный блок...
  поля: количество, цена_cost, маржа%, → цена_sales
  source: kp | added

УРОВЕНЬ 3 — Номенклатуры (наполнение проектированием)
  конкретные позиции с количествами
```

## Вопросы по реальному КП

1. Множитель "1.00" в каждой строке — что это? Коэффициент запаса?
   Курс валюты? Уточнить.

2. Группы КП фиксированные или свободные?
   (Оборудование, Холодоснабжение, Автоматизация... — это всегда так
    или менеджер создаёт группы свободно?)

3. Строки-материалы (Воздуховоды оц 720м) — раскрываются в номенклатуры
   или это уже финальная позиция? (720м воздуховода = сумма всех
   воздуховодов проекта? или отдельная закупка материалом?)

4. "Приточная установка П1 ВЕЗА" — ВЕЗА это производитель?
   Установка покупается готовой у ВЕЗА или собирается из номенклатур?

5. Маржа на уровне строки (40%) перекрывает общую (57%)?

6. Холодоснабжение П2/П3 — группы привязаны к установкам П2/П3.
   Связь между группами? (Холодоснабжение П2 относится к Приточной П2)

7. Раздел один ("Оборудование и материалы ВиК") или их несколько?
   (может быть раздел Вентиляция, раздел Электрика...?)

## УТОЧНЕНИЯ (раунд 2) — ФИНАЛЬНАЯ модель КП

### Структура — 2 уровня + раскрытие

```
ГРУППА (базовые, но можно добавлять новые)
  Оборудование, Холодоснабжение, Автоматизация,
  Теплоснабжение, Материалы... (+ свои)
    │
    ЗАПИСЬ (строка КП — верхнеуровневое описание)
      "Приточная установка П1", "Прочие клапаны",
      "Воздуховоды оц", "Наружный блок MDVO"...
        │
        раскрывается на проектировании ↓
        НОМЕНКЛАТУРЫ (одна или несколько)
```

### Запись = контейнер для номенклатур

```
Запись КП — ЧТО продали (верхний уровень)
Номенклатуры — ЧТО реально нужно (детали)

Одна запись → 1..N номенклатур:
  "Прочие клапаны" → Клапан А, Б, В... (несколько)
  "Наружный блок MDVO" → 1 номенклатура (готовое изделие)
  "Приточная П1" → Воздуховод, Вентилятор, Фильтр... (много)
```

### Связи между записями — НЕ делаем (пока)

```
Холодоснабжение П2 и Приточная П2 визуально связаны (П2),
но в модели связь не делаем. Каждая запись — в своей группе.
→ проще, без лишних связей
```

### Модель данных КП

```
kp_groups (группы — базовые + добавляемые)
  id, project_id, name, order
  ("Оборудование", "Холодоснабжение", "Материалы"...)

kp_items (записи — строки КП)
  id, group_id, name, quantity, unit,
  price_cost (себестоимость/ед), margin_percent,
  source: kp | added, order
  ("Приточная установка П1", "Прочие клапаны"...)

kp_item_positions (раскрытие — связь записи с номенклатурами)
  id, kp_item_id, position_id (FK→element_positions), quantity
  → одна запись → 1..N номенклатур
  → заполняется на этапе проектирования
```

### Связь КП ↔ подзадачи

```
Запись КП (kp_items) — это и есть корневой блок проекта.
При раскрытии в номенклатуры → они становятся
содержимым для снабжения (что закупать).

kp_item = верхнеуровневый блок
kp_item_positions = его номенклатурное наполнение
дальше эти позиции дробятся на подзадачи снабжения/работ
```

### Базовые группы (стартовый набор)

```
1. Оборудование          (установки приточные/вытяжные, циклоны, зонты)
2. Холодоснабжение       (наружные блоки, трассы, комплекты)
3. Автоматизация         (шкафы, преобразователи, датчики)
4. Теплоснабжение        (арматура, узлы, трубы)
5. Материалы             (воздуховоды, фасонка, изоляция, кабель)
6. Работы                (монтаж, ПНР — наполняется работами)
7. Документация          (возможно)
+ возможность добавить свои
```

- Раздел верхнего уровня (Оборудование и материалы ВиК) —
