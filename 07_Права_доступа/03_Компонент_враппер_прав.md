# Permission Wrapper — компонент-обёртка для прав

Универсальный компонент: оборачивает любой UI, проверяет права,
показывает/скрывает/блокирует. Права строками (ERGO.Projects.Get).

═══════════════════════════════════════════════════
ИДЕЯ
═══════════════════════════════════════════════════
```
Компонент <Can> принимает массив прав, сверяет с правами
пользователя, и по результату показывает/скрывает/блокирует.

<Can permissions={["ERGO.Supply.Approve"]}>
  <button>Согласовать</button>
</Can>

Есть право → показывает. Нет → скрывает (или блокирует).
```

═══════════════════════════════════════════════════
ПРОПСЫ КОМПОНЕНТА
═══════════════════════════════════════════════════
```
<Can
  permissions={["ERGO.Supply.Approve"]}  — массив требуемых прав
  require="any"      — "any" (хотя бы одно) | "all" (все)
  mode="hide"        — "hide" (скрыть) | "disable" (заблокировать)
  fallback={null}    — что показать если нет прав (опц.)
>
  {children}
</Can>

По умолчанию: require="any", mode="hide".
```

═══════════════════════════════════════════════════
РЕАЛИЗАЦИЯ (React)
═══════════════════════════════════════════════════
```jsx
// Права пользователя — из контекста (получены при логине)
const { userPermissions } = usePermissions();
// userPermissions = ["ERGO.Projects.Get", "ERGO.Supply.Request", ...]

function Can({ permissions, require = "any", mode = "hide",
              fallback = null, children }) {
  const has = require === "all"
    ? permissions.every(p => userPermissions.includes(p))
    : permissions.some(p => userPermissions.includes(p));

  if (has) return children;

  // нет прав:
  if (mode === "disable") {
    // клонируем children с disabled
    return React.cloneElement(children, { disabled: true });
  }
  return fallback;  // hide (по умолчанию null)
}
```

═══════════════════════════════════════════════════
ПРИМЕНЕНИЕ — на любом уровне
═══════════════════════════════════════════════════
```jsx
// КНОПКА действия
<Can permissions={["ERGO.Supply.Approve"]}>
  <button>Согласовать</button>
</Can>

// РАЗДЕЛ МЕНЮ (скрыть у кого нет доступа)
<Can permissions={["ERGO.Admin.Users"]}>
  <MenuItem to="/users">Пользователи</MenuItem>
</Can>

// КОЛОНКА/ЧАСТЬ СПИСКА (прораб не видит цены)
<Can permissions={["ERGO.Prices.View"]}>
  <TableColumn field="price" header="Цена" />
</Can>

// ЗАБЛОКИРОВАТЬ (видно, но неактивно)
<Can permissions={["ERGO.Payment.Pay"]} mode="disable">
  <button>Оплатить</button>
</Can>

// НЕСКОЛЬКО ПРАВ (любое из)
<Can permissions={["ERGO.Supply.Edit", "ERGO.Admin.All"]} require="any">
  <EditButton />
</Can>

// FALLBACK (показать заглушку)
<Can permissions={["ERGO.Prices.View"]} fallback={<span>—</span>}>
  <PriceValue />
</Can>
```

═══════════════════════════════════════════════════
ЛОГИКА МАССИВА ПРАВ
═══════════════════════════════════════════════════
```
require="any" (по умолчанию):
  показать если есть ХОТЯ БЫ ОДНО право из массива
  пример: [Supply.Edit, Admin.All] — редактор ИЛИ админ

require="all":
  показать только если есть ВСЕ права
  пример: [Project.View, Prices.View] — и проект, и цены

Гибко под разные случаи.
```

═══════════════════════════════════════════════════
ПОЛУЧЕНИЕ ПРАВ ПОЛЬЗОВАТЕЛЯ
═══════════════════════════════════════════════════
```
При логине → бэк отдаёт permissions пользователя:
  GET /api/me → { user, permissions: ["ERGO.Projects.Get", ...] }
  (у тебя уже есть — список методов роли)

Фронт кладёт в контекст (PermissionsProvider):
  <PermissionsProvider value={permissions}>
    <App />
  </PermissionsProvider>

<Can> берёт из контекста, сверяет.

→ изменил права роли на бэке → пользователь перелогинился →
  новый список → <Can> пересчитал ВСЁ автоматически
  (код фронта не трогали)
```

═══════════════════════════════════════════════════
ВАЖНО — фронт не замена защите
═══════════════════════════════════════════════════
```
<Can> — UX (скрыть/заблокировать неудобное).
БЭК всё равно проверяет КАЖДЫЙ запрос (реальная защита).

Скрыл кнопку "Оплатить" → но если вызовут API →
бэк проверит право ERGO.Payment.Pay → 403 если нет.

<Can> убирает лишнее из виду, бэк защищает по-настоящему.
```

═══════════════════════════════════════════════════
СВЯЗЬ С УРОВНЯМИ (из файла 02)
═══════════════════════════════════════════════════
```
<Can> — это УРОВЕНЬ 1 доведённый до удобства:
  права с бэка + переиспользуемый компонент проверки
  (вместо { can('X') && ... } везде — единый <Can>)

Для действий этапов можно комбинировать с Уровнем 2:
  бэк отдаёт actions[] с enabled →
  <Can> не нужен, кнопки уже с бэка

Гибрид:
  статичные права (меню, колонки, разделы) → <Can>
  динамичные действия (зависят от статуса+роли) → actions[] с бэка
```
