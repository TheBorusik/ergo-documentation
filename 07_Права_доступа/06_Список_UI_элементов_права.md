# Список UI-элементов под права (ЖИВОЙ документ)

Элементы UI которые должны контролироваться правами.
Формат ключа: app.screen.element (для ERP: erp.*)

> ⚠️ ЖИВОЙ ДОКУМЕНТ — дополняется по мере проектирования UI.
> Новый экран → добавить его элементы сюда.
> Финализация прав — когда UI устоится.
> Занесено из ГОТОВОГО UI (карточка, этапы, навигация).

Колонки: ключ | элемент | право | mode | КОМУ ДОСТУПНО

КАК ЧИТАТЬ:
  "КОМУ ДОСТУПНО" = роль у которой ЕСТЬ право (кому показывать/разрешать)
  "mode" = что делать у кого права НЕТ: hide (скрыть) | disable (заблокир.)

Пример: erp.supply.create_task | ... | hide | снабженец
  → кнопка ДОСТУПНА снабженцу
  → у остальных (не снабженец) СКРЫТА (hide)

НЕ значит "скрыто от снабженца" — наоборот, снабженцу доступно!

ВАЖНО: колонка "КОМУ ДОСТУПНО" — СПРАВОЧНАЯ (для нас, чтения),
  это НЕ поле в БД. В БД компонент знает только ПРАВО, а не роль.
  Связь: компонент → ПРАВО → роль (право посредник).
  "Кому доступно" — вывод: у какой роли есть это право.
  Отдельно эта колонка нигде не хранится — вычисляется из
  (роль→права) + (компонент→право).

═══════════════════════════════════════════════════
ЛЕГЕНДА
═══════════════════════════════════════════════════
```
★ — критичные разграничения (согласование/оплата — разные роли)
mode: hide (скрыть) | disable (заблокировать, видно но неактивно)
"кто" — предполагаемая роль (уточнить)

Права — предварительные названия (ERP.Category.Action).
Реальные коды — как в твоей системе (ERGO.* или ERP.*).
```

ЧТО ЕЩЁ ДОБАВИТСЯ (по мере UI)
═══════════════════════════════════════════════════
```
- Дашборд (виджеты, метрики — кто что видит)
- Список проектов (фильтры, действия)
- Номенклатура (создание, редактирование, блокировка)
- Поставщики (карточка, цены, каталог)
- Окно выбора (доступность по этапу — уже есть в логике)
- Формы справочников (детально)
- Настройки, профиль

→ каждый новый экран → его элементы сюда
```

═══════════════════════════════════════════════════
КЛЮЧЕВЫЕ РАЗГРАНИЧЕНИЯ (из готового UI)
═══════════════════════════════════════════════════
```
СОГЛАСОВАНИЕ — только менеджер (не снабженец!)
ОПЛАТА — только бухгалтер (не снабженец!)
СОЗДАНИЕ ПРОЕКТА — менеджер
ВЫИГРЫШ/ПРОИГРЫШ — менеджер
РЕДАКТИРОВАНИЕ КП — менеджер
СПЕЦИФИКАЦИИ — инженер
ЗАКУПКА — снабженец
РАБОТЫ — прораб
ДОКУМЕНТАЦИЯ — сотрудник
СПРАВОЧНИКИ/АДМИНКА — админ
ЦЕНЫ (видимость) — снабженец/менеджер (прораб — вопрос)
СОЗДАНИЕ ORDER / ОПЛАТА — только бухгалтер (этап Закупки)
РАЗДЕЛ ЗАКУПКИ — бухгалтер, менеджер, руководитель
```

ВОПРОСЫ

2. Кто видит блок финансов проекта? (менеджер? бухгалтер?)

4. Видимость разделов — все видят все этапы или свои?
5. Названия прав — ERP.* или как в твоей системе?
